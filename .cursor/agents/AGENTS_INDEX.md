## Обзор агентов для оркестрации

### Приоритетные агенты для текущего проекта (система парковки)

- **agents-orchestrator**: Центральный менеджер пайплайна, который orchestrat'ит работу всех остальных агентов, следит за фазами, quality‑gates и отчётностью.
- **product-manager**: Помогает формулировать и уточнять требования к системе парковки, приоритизировать фичи и связывать их с бизнес‑целями.
- **workflow-architect**: Строит детальные пользовательские и сервисные потоки (заезд, оплата, штрафы, абонементы) до уровня, удобного для реализации и тестов.
- **backend-architect**: Проектирует доменную модель парковки, API, интеграции с внешними системами и стратегию масштабирования/надёжности.
- **frontend-developer**: Помогает быстро прототипировать и дорабатывать UI (веб‑кабинеты, панели операторов, экраны терминалов/киосков).
- **ux-architect**: Структурирует интерфейсы и навигацию для разных ролей (водитель, оператор, администратор) с упором на понятность.
- **ux-researcher**: Формирует сценарии исследований и юзабилити‑тестов, чтобы проверить удобство критичных операций (оплата, выезд, обжалование).
- **analytics-reporter**: Настраивает и описывает отчётность по загрузке парковок, выручке, штрафам и эффективности тарифов.
- **data-engineer**: Проектирует пайплайны данных (транзакции, события въезда/выезда, сенсоры) для последующей аналитики и ML‑кейсов.
- **security-engineer**: Оценивает угрозы (мошенничество, обход оплат, доступ к персональным данным), проводит secure‑ревью и рекомендации по защите.
- **sre-site-reliability-engineer**: Определяет SLO для критичных путей (оплата, шлагбаум), настраивает мониторинг, алерты и подход к инцидентам.
- **accessibility-auditor**: Проверяет, что веб‑интерфейсы и панели самообслуживания доступны пользователям с ограниченными возможностями.
- **technical-writer**: Оформляет требования, спецификации и руководства по системе парковки в виде понятной, непротиворечивой и поддерживаемой документации.
- **software-architect**: Помогает принимать архитектурные решения на уровне всей системы (микросервисы vs монолит, шина данных, интеграции).
- **project-shepherd**: Разбивает требования по парковочному проекту на задачи, синхронизирует команды и следит за сроками.
