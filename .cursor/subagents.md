# Доступные `subagent_type`

## subagent_type

- `generalPurpose`: Универсальный агент для поиска по коду/докам, исследования и выполнения многошаговых задач.
- `explore`: Быстрое исследование кодовой базы, поиск файлов/символов по паттернам и ответ на вопросы “где/что реализовано”.
- `shell`: Специалист по запуску bash-команд, Git и другим операциям в терминале.
- `browser-use`: Браузерная автоматизация для тестирования веб-приложений (переходы, ввод, клики, проверки UI).
- `architecture-view-linker`: Помогает сопоставлять требования/интерфейсы с архитектурными компонентами (быстрые архитектурные связи).
- `demo-days-presentations`: Подготовка структурированных и ярких презентаций/сценариев для внутренних демо-дней.
- `doc-to-markdown`: Конвертация загруженных документов (PDF/DOCX/изображения и т.п.) в структурированный Markdown.
- `fr-parking-analyst`: Анализирует требования и помогает спроектировать/дополнить функциональность системы управления парковкой по UC и концептуальной модели.
- `frontend-ux-ui`: Создание и доработка UX/UI интерактивных макетов (с учетом Nunjucks/SCSS и практик UX).
- `git-workflow`: Рекомендации по Git-workflow и помощь в корректных коммитах/ветках/PR.
- `infosec`: Анализирует угрозы, утечки, атаки и практики безопасности/соответствия требованиям (в т.ч. 152-ФЗ).
- `meta-instructor`: Создает инструкции и файлы-описания для других субагентов.
- `requirements-spec-maintainer`: Поддерживает формальные спецификации в `docs/specs/*.md`, учитывая проектные правила и артефакты анализа.
- `systems-analyst`: Анализ требований и подготовка артефактов системного анализа и черновиков формальных требований.
- `team-lead-orchestrator`: Подбирает нужные субагенты и формирует план/оркестрацию работ для достижения результата.
- `traceability-matrix-builder`: Создает и поддерживает матрицы трассировки требований к источникам, артефактам и интерфейсам.
- `wireframe-from-artifacts`: Генерирует wireframe-макеты (HTML+CSS) по артефактам системного анализа и требованиям.
