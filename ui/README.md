# Wireframe — цифровая платформа парковки

Интерактивный wireframe-макет по артефактам (`docs/artifacts/`) и отчёту проверки (`docs/specs/Отчёт-проверки-wireframe-по-требованиям-качества-UI.md`). Файлы распределены по папкам продуктов (Контекстная диаграмма, Карта навигации).

## Точка входа

Откройте **`index.html`** в корне `ui/` — выбор раздела: сайт-визитка, вход клиента, вход админа, вход охранника КПП.

## Структура по продуктам

| Папка | Продукт | Содержимое |
|-------|---------|------------|
| **site/** | Business Site (сайт-визитка) | Главная, контакты (адрес и въезды), тарифы, карта мест, правила парковки, регистрация, вход клиента |
| **client/** | Consumer App (ЛК клиента) | Главная ЛК, бронирование, оплата (через внешний сервис), договоры, история сессий, список ТС, профиль |
| **admin/** | Admin Panel (ЛК администратора) | Дашборд, секторы и ПМ, тарифы, договоры, задолженности, профили клиентов, профили админов, обращения, аналитика |
| **guard/** | Рабочий интерфейс КПП | Рабочий интерфейс охранника (вход через `site/staff-login.html`), проверка доступа (поиск по ГРЗ / телефону / ID бронирования), результат, действия |

## Файлы в корне

- **wireframe.css** — общие стили (WF-05 skip-link, WF-06 фокус, WF-07 блок ошибок)
- **index.html** — точка входа с ссылками на продукты

## Мобильная версия и адаптация ЛК клиента

- В ЛК клиента (`layouts/lk.njk`, страницы в `client/` и `templates/pages/*` для ЛК) реализован **мобильный режим**:
  - в футере есть переключатель «для компьютера / для телефона», который управляет атрибутом `data-view="mobile"` на `<body>`;
  - на мобильных экранах и при включённом мобильном режиме:
    - навигация ЛК сворачивается в бургер‑меню (кнопка в шапке + клик по названию текущего раздела),
    - фильтры (`.sessions-filters`) перестраиваются в несколько строк и тянутся на ширину экрана,
    - модальные окна ограничены по ширине (примерно экран смартфона) и центрируются.

## Реализованные задачи отчёта (WF-01–WF-12)

- WF-01: шаг оплаты после брони → `client/payment-external.html`
- WF-02: тариф и «Итого к оплате» в форме бронирования → `client/bookings.html`
- WF-03: блоки поиска на тарифах, договорах, клиентах, бронированиях, истории сессий
- WF-04: поиск по ГРЗ / телефону / ID в КПП → `guard/index.html`
- WF-05: ссылка «Перейти к содержимому», `id="main-content"` у `main` — во всех страницах
- WF-06: стили `:focus-visible` и `.wf-error` в `wireframe.css`
- WF-07: плейсхолдеры сообщений об ошибках на формах (вход, регистрация, бронирование, КПП, админ, охранник)
- WF-08: хлебные крошки на вложенных и публичных страницах
- WF-09: блок «Адрес и точки въезда» → `site/contacts.html`
- WF-10: раздел «Правила парковки» → `site/rules.html`
- WF-11: раздел «Обращения» с поиском → `admin/requests.html`
- WF-12: поиск/фильтр по сектору и типу места на карте → `site/parking-map.html`

## Навигация

- Из точки входа `ui/index.html`: ссылки на `site/index.html`, `site/login.html` и `site/staff-login.html`
- Из `site/staff-login.html`: маршрутизация по логину — `admin` → `../admin/index.html`, `operator` → `../guard/index.html`
- Из ЛК клиента: «На сайт» → `../site/index.html`, «Выход» → `../site/login.html`
- Из ЛК админа и КПП: «Выход» ведёт на общий экран входа для сотрудников `../site/staff-login.html`
