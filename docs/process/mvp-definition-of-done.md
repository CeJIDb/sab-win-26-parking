## Назначение

Документ фиксирует Definition of Done для MVP парковочной платформы.

Этот DoD описывает, какой функциональный минимум должен быть реализован и согласован, чтобы первую поставку можно было считать завершенной на уровне бизнес-сценариев.

## Общий критерий

Система переводит операции парковки с ручного Excel-учета в цифровой формат, обеспечивая каждому типу пользователя возможность выполнять свои задачи самостоятельно через личный кабинет или интерфейс КПП.

## Критерии по областям

### Работа с тарифами

Клиент может самостоятельно ознакомиться с актуальными тарифами через личный кабинет без обращения к персоналу.

Владелец может создавать, редактировать и архивировать тарифы в едином интерфейсе.

### Доступ к сервису

Физическое лицо может самостоятельно зарегистрироваться и войти в личный кабинет.

Корпоративный клиент получает доступ через управляющего.

Владелец может управлять учетными записями и ролями сотрудников.

### Автопарк

Любой клиент может добавить, изменить или удалить транспортное средство в своем профиле.

### Договор

Клиент-физлицо может самостоятельно оформить договор, подписать его и управлять им без визита в офис.

В область управления договором входят продление и расторжение.

Управляющий может оформлять договоры со стороны парковки.

### Оплата

Клиент-физлицо может оплатить парковку онлайн или через платежный терминал.

Система автоматически фиксирует оплату и обновляет статус договора или парковочной сессии.

Управляющий видит финансовый статус без ручной сверки таблиц.

### Уведомления

Все клиенты получают своевременные уведомления о событиях, относящихся к их зоне ответственности.

Примеры таких событий: истечение договора, факт оплаты, изменение статуса места.

### Система доступа

Статусы личного кабинета, договоров и оплат автоматически синхронизируются с системой доступа.

Охранник видит актуальную информацию на своем экране.

Въезд и выезд фиксируются автоматически.

Ручной ввод доступен как резервный сценарий при сбое.

### Управление машиноместами

Управляющий может настроить схему парковки и задать лимиты по зонам.

Управляющий может блокировать места на период работ и закреплять места за договорами.

Охранник видит актуальные статусы мест в реальном времени.

## Сквозные критерии

Каждый клиент имеет доступ только к функциям своего уровня в соответствии со своей ролью.

Данные между модулями "личный кабинет -> договор -> оплата -> доступ" синхронизируются без ручного вмешательства.

Система работает как единый источник правды вместо независимых Excel-файлов.
