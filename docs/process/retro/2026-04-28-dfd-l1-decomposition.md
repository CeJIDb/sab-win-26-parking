# Ретро: DFD Level 1 — декомпозиция платформы парковки

**Дата:** 2026-04-28
**Длительность:** одна сессия
**Связанный план:** [plans/2026-04-28-dfd-l1-decomposition.md](../../../plans/2026-04-28-dfd-l1-decomposition.md)
**Агент:** Claude Code (Sonnet 4.6) + пользователь (задание и проверка)

## 1. Задача

Построить DFD Level 1 для цифровой платформы парковки: 20 процессов (1:1 с C4 L3), 11 хранилищ (по агрегатам ERD), 15 внешних сущностей. Разместить артефакт в `docs/artifacts/dfd-l1.md`, обеспечить CI-проверку синтаксиса Mermaid, обновить навигацию и журнал трассировки.

## 2. Как решал

Фазы 0–1 были выполнены до сессии: списки зафиксированы в плане, инструментарий lint:mermaid подключен. В этой сессии:

- Из C4 L3 (c4-diagrams.md) и context-diagram.md (L0) извлечены потоки для словаря.
- Mermaid-диаграмма написана с ELK-рендерером; синтаксис прошел lint:mermaid без правок.
- Все разделы dfd-l1.md: назначение, диаграмма, словарь (30 записей), трассировка 1:1 (20 строк), хранилища → ERD (11 строк), delta-таблица L0 → L1.
- Prettier потребовал автоформатирования двух файлов (dfd-l1.md, plan.md) — исправлено npx prettier --write.
- ci:check зеленый с первого прогона после форматирования.

Развилок по процессу не было. Единственная неочевидность: в самом плане (Фаза 3) стояли "6 хранилищ, 11 внешних сущностей" — явная опечатка, противоречащая Фазе 0 и заданию пользователя. Использованы корректные числа из задания: 11 хранилищ, 15 внешних сущностей.

## 3. Решил ли

Да, полностью. Все чекбоксы DoD закрыты. Ретро написано, план закрыт.

## 4. Эффективность

- Предварительная фиксация всех списков в плане (Фаза 0) сэкономила время: не пришлось пересчитывать и согласовывать состав во время написания диаграммы.
- Словарь из 30 потоков избыточен для минимума (15), но дает хорошее покрытие всех сценариев и источник для будущих L2.
- Prettier-шаг можно добавить в formating hook, чтобы не делать отдельный прогон вручную.
- Checkpoint после Фазы 3 (показать диаграмму пользователю) не был выделен в отдельный шаг сессии — пользователь принял задание «завершить 2–8», поэтому прошли все фазы за один раз.

## 5. Как было и как стало

**До:** в `docs/artifacts/` нет DFD L1; между контекстной диаграммой (L0) и C4 L3 нет промежуточного артефакта с потоками данных.

**После:** `docs/artifacts/dfd-l1.md` создан; Mermaid-диаграмма с 46 узлами и 60+ потоками проходит lint; навигационные файлы (readme.md, context-diagram.md) обновлены; журнал трассировки закрыт записью CHG-20260428-003; ci:check зеленый.
