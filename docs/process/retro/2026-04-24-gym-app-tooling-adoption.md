# Ретро: перенос инфраструктуры из gym-app

**Дата:** 2026-04-24
**Длительность:** ~4 часа (в рамках бюджета плана)
**Агент:** Claude Code (смешанная сессия: агент + пользователь)

## 1. Задача

Адаптировать в репозиторий parking инструменты и процессные артефакты из соседнего проекта gym-app: защитный контур Claude Code, единый конфиг форматирования (Prettier + markdownlint-cli2), папку планов и ретроспектив, доработку скриптов атомарных коммитов и интеграцию Playwright MCP.

## 2. Как решал

Работал последовательно по блокам A → B → C → D.

Блок A (`.claude/settings.json`, `CLAUDE.md`) — без неожиданностей, прямой порт с адаптацией стека.

Блок B оказался сложнее: markdownlint-cli2 на baseline показал массовые нарушения MD022/MD032/MD028
в существующих документах. Вместо немедленной чистки всех файлов выбрал подход «сознательного
ослабления» — добавил обоснованные исключения в `.markdownlint.jsonc` и точечные ignores в
`.markdownlint-cli2.jsonc`, чтобы получить зеленый CI без массового diff.

Блок C дал `validate-plans.mjs` из gym-app почти без правок.

Блок D потребовал полного переписывания `atomic-commit.mjs`: движок gym-app существенно лучше
(статусные коммиты A/M/D/R, корректный разбор через `--porcelain=v1 -z`, `--staged-only`),
поэтому взяли его за основу и настроили бакеты под parking.

Отклонение: `CONTRIBUTING.md` и `README.md` не трогали (желательное, но не блокирующее по DoD плана). `atomic-commit.sh` (bash-fallback) не синхронизирован с новой `.mjs`-логикой — оставлен на следующий план.

## 3. Решил ли

Да — полностью в рамках одной сессии. Все 9 пунктов аудита закрыты, все 6 фаз плана выполнены, `npm run ci:check` зеленый на 90 markdown-файлах. Открытые пункты явно выведены в «Остается» плана и не блокируют PR.

## 4. Эффективность

**Что сработало:**

- Детальный план с тайминг-таблицей помог не расплыться: знали, что пропустить без риска (желательные пункты).
- Подход «baseline сначала, чистка потом» для markdownlint избежал тысячестрочного diff в первом PR.
- Взять готовый движок atomic-commit из gym-app вместо точечных правок parking-версии — резко сократило время блока D.

**Что стоило сделать лучше:**

- Заранее прогнать `markdownlint-cli2` на сухую, до настройки конфигов — сэкономило бы время на итерациях с ослаблением правил.
- Синхронизацию `atomic-commit.sh` стоило включить в scope: теперь есть drift между `.sh` и `.mjs`, который будет накапливаться.

## 5. Как было и как стало

**До:** репозиторий не имел защитного контура для Claude Code, использовал только самописный `check-markdown.mjs`, не имел единого конфига форматирования, папки планов и регламента ретроспектив.

**После:** `.claude/settings.json` с deny-правилами, `CLAUDE.md` с картой репозитория, Prettier + markdownlint-cli2 в devDependencies с конфигами, `plans/` + `docs/process/retro/` с регламентами, переработанный `atomic-commit.mjs` на движке gym-app, `.mcp.json` с Playwright MCP.
