## **Ход встречи**

    ### **BPMN**

        #### **Оплата**

            - Процесс зафиксирован корректно.

            - Дальнейшая детализация не требуется на текущем уровне.

        #### Заключение договора

            - **Физические лица и юридические лица — два разных бизнес-процесса.**

            - Рекомендация: **разбить на 2 отдельные BPMN-диаграммы** для читаемости.

            - С точки зрения нотации — корректно, проблема именно в перегруженности.

        #### Выезд

            - **Идентификацию оформить как подпроцесс**.

            - Не углубляться в мелкие операционные задачи (оставить уровень событий/решений).

    ### Opportunity Canvas

        #### Формулировки

            - Исключить жаргон.

            - Переформулировать:

                - «финансовые потери из-за простоя» → **«недополученный доход»**.

            - Внедрить **выделение ключевых проблем жирным шрифтом** как хорошую практику.

    ### Статус артефактов AS-IS

        **Все артефакты согласованы** с учётом оставленных заказчиком комментариев:

            - Диграмма **Event Storming** для процесса предоставления парковочного места

            - Диграммы **BPMN**:

                - Предоставление парковочного места

                - Идентификация клиента

                - Оплата

                - Заключение договора

                - Поиск парковочного места

                - Выезд с парковки

            - **МПО UML Class Diagram**

            - **UML State Machine** Договор с физлицом

            - **Opportunity Canvas (левая часть)**

                ### Impact Map

        **Цель проекта:** увеличить долю клиентов, использующих онлайн-сервисы, до **80% от общего количества клиентов на дату запуска решения**. Отсчет начинается с момента запуска, срок достижения — 6 месяцев.

        #### Акторы

            - **Основные акторы:**

                - юрлица

                - физлица

                - охранники

                - управляющий

                - владелец

            - **Примечание:**

                - **Масштабирование парковки** — **один управляющий управляет несколькими парковками**.

                - **Подрядчики** принимают минимальное участие, не являются ключевым актором.

        #### Воздействия

            - Управление ТС:

                - **Подавать и редактировать список ТС — и для ФЛ, и для ЮЛ**.

                - Объединить карточки.

            - Информирование клиентов  — сделать **более обобщённым**, без детализации каналов.

            - Выставление счетов — **не в первом приоритете**.

            - Инциденты — переформулировать в **«создание обращения / поддержка / обратная связь»**.

            - Бронирование **— онлайн для физических лиц**.

            - Ключевой принцип **— въезд и выезд без взаимодействия с персоналом парковки**.

            - Аналитика:

                - Объединить:

                    - «метрики для маркетинга»

                    - «данные для аналитики».

            - «Сверка поступивших оплат» → **управление доступом**.

            - Аудит **— история использования парковки**. Объединить с историей для ФЛ.

            - **Онлайн-оплата — обязательная возможность.**

            - Обратная связь — объединить с обращениями.

            - Управление задолженностями:

                - автоматические напоминания,

                - возможность оплаты,

                - графики платежей,

                - уведомления.

            - Договоры в онлайн: **заключить → отслеживать статус → продлить**.

        #### Поставка / Решения

            - **Решения для авторизованных клиентов:**

                - Личный кабинет клиента.

                - Онлайн-бронирование/выбор места

                - Самообслуживание по спискам ТС

                - Конструктор договоров— требуется.

            - **Решения для неавторизованных пользователей:**

                - Сайт-визитка (неавторизованный доступ)

                - Геосервисы

            - **Решения для управляющего и персонала парковки:**

                - CRM

                - Панель администратора

                    - Управление:

                        - шаблонами договоров,

                        - тарифами,

                        - сотрудниками.

                - Модуль обращений/поддержки

                - Отчетность/аналитика (для владельца и управляющего)

            - **Интеграции**

                - Биллинг/платежи

                    - Полноценный документооборот — **только для юрлиц**.

                    - Для физлиц — упрощённая модель.

                - ЭДО:

                    - Риск: разные провайдеры или отсутствие ЭДО у клиентов.

                    - Требуется **универсальное решение**.

                - Контроль доступа (СКУД/проезд):

                    - механизм бесконтактного въезда/выезда + запасной сценарий на случай отказа системы

        #### Приоритеты

            - **Высший приоритет:** Все, что устраняет очереди и ручную работу охранников. Ключевая функция — **бесконтактный въезд/выезд**.

            - **Обязательный функционал:** Личный кабинет для клиентов (заключение, отслеживание, оплата), панель администратора (CRM, управление тарифами, шаблонами, сотрудниками), QR-код на КПП.

            - **Требует проработки:** 

                - Биллинг, интеграция с ЭДО (учитывая разнообразие систем у клиентов), размещение на геосервисах, сайт-визитка.

                - Сохранение взаимодействия с 20% нецифровых клиентов, которые по какой-либо причине не могут полноценно использовать онлайн-сервисы (например оставить QR-код на въезде), чтобы не потерять их, как клиентов. 

        #### **Распределение ответственности**

            - **Владелец:** остается ответственным за **подписание договоров с юридическими лицами**.

            - **Управляющий**: 

                - Все остальные операционные вопросы.

                - Будет использовать систему для анализа и принятия стратегических решений.
