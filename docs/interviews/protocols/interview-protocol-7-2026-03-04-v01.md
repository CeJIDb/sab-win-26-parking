## **Ход встречи**

**1. Введение и организационные моменты**

- Встреча началась с опозданием участников по техническим причинам. Секретарь ([malinkaemail-blip](https://github.com/malinkaemail-blip)) и хронометрист ([Denis333admin](https://github.com/Denis333admin)) присоединились позднее из-за форс-мажора.

- [skifup](https://github.com/skifup) уточнил у Заказчика формат дальнейшего взаимодействия в связи с завершением графика интервью. Заказчик подтвердила, что на этапе технического проектирования основная коммуникация будет перенесена в чат с Ментором. Встречи с заказчиком возможны по необходимости для уточнения бизнес-требований.

**2. Схема навигации экранов**

- [CeJIDb](https://github.com/CeJIDb) презентовал обновленную схему. Основные изменения: детализация переходов, вынос быстрых действий (оплата, создание договора) на главный экран, группировка редко используемых функций.

- **Обсуждение:**
  - Уточнены права доступа для ролей «Владелец» и «Управляющий». Решено, что функционал у них одинаков, но владелец может создавать учетные записи других администраторов.

  - Поднят вопрос о необходимости функционала «продление договора». Заказчик уточнила, что он нужен, но только для управляющего.

  - Обсуждены способы авторизации: решено остановиться на VK ID и Яндекс ID. Для двухфакторной аутентификации сотрудников выбран TOTP (генератор кодов) вместо SMS.

**3. Интеграция с СКУД**

- Заказчик запросила визуализацию сценарий работы охранника при интеграции с системой распознавания.

- **Решение:** Команда взяла в работу подготовку примеров интерфейсов (UI) для наглядной демонстрации логики работы.

**4. Моделирование процессов (Event Storming To-Be)**

- Была представлена декомпозицию процессов для распределения ответственности внутри команды.

- **Итог обсуждения:** Утвержден список из 5 ключевых подпроцессов для детальной проработки со стороны команды:
  1. Регистрация клиента,

  2. Краткосрочное бронирование (с предварительным бронированием и без него),

  3. Долгосрочное бронирование,

  4. Оплата,

  5. Проверка права доступа.

- Отмечено, что ментор (Ментор) привлек эксперта (Татьяну) для помощи в моделировании, встреча с ней назначена на понедельник.

**5. Блок: Управление клиентами (Черный список и должники)**

- Поднят вопрос о разграничении статусов «должник» и «черный список».

- **Принято решение:**
  - Статусы разделены.

  - Снятие статуса «должник» (приостановка) происходит автоматически после оплаты.

  - Снятие статуса «черный список» (запрет) — только вручную уполномоченным сотрудником с возможностью оставить комментарий.

**6. Заключительная часть**

- Подведены итоги по каждому блоку. Согласовано, что часть артефактов будет утверждена асинхронно в чате.

- Будет запланирована следующая встреча для финализации Event Storming To-Be.
