# Индекс постановок задач на разработку

Документы этого раздела используются для постановки задач на разработку на основе требований и архитектурного контекста проекта.

## Содержание

- [_template-dev-task.md](_template-dev-task.md) — шаблон постановки задачи на разработку.
- [example-task/example-task.md](example-task/example-task.md) — пример постановки задачи, перенесенный из изображения.
- [uc-3-1-add-vehicle.md](uc-3-1-add-vehicle.md) — постановка задачи по `UC-3.1` «Добавить ТС».

## Правила ведения

- Каждая постановка задачи должна ссылаться на источник требований: `docs/artifacts/*`, `docs/specs/*`, `docs/architecture/*` или `docs/interviews/*`.
- Формулируйте границы задачи явно: что входит в объем, а что не входит.
- Для новых файлов используйте lowercase `kebab-case`.
- Если постановка задачи приводит к содержательным изменениям требований или трассировки, обновляйте связанные документы в `docs/specs/` и `docs/process/traceability-matrix-log.md`.
