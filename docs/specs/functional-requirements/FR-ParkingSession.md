## КлиентФЛ

- ПарковочнаяСессия.ПросмотретьАктивную

  Система должна позволять пользователю с ролью КлиентФЛ просматривать активную для его ТС парковочную сессию со следующими атрибутами:

  - идентификатор
  - номерБронирования
  - времяВъезда
  - `[КПП]:наименование`
  - способДопуска
  - статус

- ПарковочнаяСессия.ПросмотретьСписок

  Система должна позволять пользователю с ролью КлиентФЛ просматривать список объектов Парковочная сессия, связанных с его ТС и Бронированиями, с отображением для каждой парковочной сессии следующих атрибутов:

  - идентификатор
  - номерБронирования
  - времяВъезда
  - времяВыезда
  - длительность
  - статус

## Системы

Данные для автоматического создания и завершения парковочной сессии поступают из объекта СобытиеСКУД.

- ПарковочнаяСессия.Создать.СД

  Система должна создать объект Парковочная сессия, заполняя атрибуты по следующему алгоритму:

  - идентификатор - согласно ФТ `ПарковочнаяСессия.Создать.СД.идентификатор`
  - бронированиеИд = идентификатор активного Бронирования для данного ТС и парковки, определённого в UC-12.1 и UC-12.2
  - времяВъезда = `[СобытиеСКУД]:датаВремяСобытия`
  - кппВъездаИд = `[СобытиеСКУД]:кппИд`
  - способДопуска = «автоматически»
  - статус = «активна»

- ПарковочнаяСессия.Создать.СД.статус

  Система должна изменять `[ПарковочнаяСессия]:статус` по следующему алгоритму:

  - при создании парковочной сессии в UC-12.4 `[ПарковочнаяСессия]:статус` = «активна»

- ПарковочнаяСессия.Создать.СД.идентификатор

  Система должна изменять `[ПарковочнаяСессия]:идентификатор` по следующему алгоритму:

  - `[ПарковочнаяСессия]:идентификатор` = max(`[ПарковочнаяСессия]:идентификатор`) + 1

- ПарковочнаяСессия.Завершить.СД

  Система должна изменять объект Парковочная сессия по следующему алгоритму:

  - времяВыезда = `[СобытиеСКУД]:датаВремяСобытия`
  - длительность - согласно ФТ `ПарковочнаяСессия.РассчитатьДлительность.СД.длительность`
  - статус - согласно ФТ `ПарковочнаяСессия.Завершить.СД.статус`

- ПарковочнаяСессия.Завершить.СД.статус

  Система должна изменять `[ПарковочнаяСессия]:статус` по следующему алгоритму:

  - при фиксации выезда ТС в UC-12.8 `[ПарковочнаяСессия]:статус` = «завершена»

- ПарковочнаяСессия.РассчитатьДлительность.СД.длительность

  Система должна изменять `[ПарковочнаяСессия]:длительность` по следующему алгоритму:

  - если `[ПарковочнаяСессия]:статус` = «активна», то `[ПарковочнаяСессия]:длительность` = округлённая до минут разница между текущим системным временем и временемВъезда
  - если `[ПарковочнаяСессия]:статус` = «завершена», то `[ПарковочнаяСессия]:длительность` = округлённая до минут разница между временемВыезда и временемВъезда
  - если времяВыезда < времяВъезда для завершённой парковочной сессии или текущаяДатаВремя < времяВъезда для активной парковочной сессии, система должна отклонять операцию расчёта длительности и фиксировать ошибку в журнале аудита, не изменяя длительность
