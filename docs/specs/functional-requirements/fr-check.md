## Функциональные требования: объект Чек

### Входные UC

- UC-10.6 «Получить чек»
- UC-10.7 «Просмотреть список оплат»
- UC-10.9 «Просмотреть задолженность в админке»
- UC-10.x «Оплата» (формирование и фискализация чеков)

### Кандидаты ФТ (иерархия Роль.Объект.Действие)

### Реестр идентификаторов требований (FR-*)

Обоснование: для трассируемости и верифицируемости каждое требование из списка кандидатов получает уникальный идентификатор формата `FR-*`.

- **FR-CHECK-001** `Чек.ПросмотретьСписок`
- **FR-CHECK-002** `Чек.Отчет`
- **FR-CHECK-003** `Чек.Создать.СД`
- **FR-CHECK-004** `Чек.Изменить.СД.статусФискализации`
- **FR-CHECK-005** `Чек.ПовторноОтправить.СД`

## КлиентФЛ

- Чек.ПросмотретьСписок

  Система должна предоставлять пользователю с ролью КлиентФЛ возможность просматривать список своих чеков с атрибутами:

  - идентификатор
  - `идПлатежа`
  - номерФискальный
  - датаВремяЧека
  - статусФискализации
  - сумма

  и фильтрацией по периоду и статусу.

  Перечень данных внутри каждой группы:

  - `идПлатежа`
  - номерФискальный

  - статусФискализации
  - сумма
  - `идПользователя`
  - `идСотрудника`

## Системы

- Чек.Создать.СД

  Система должна при успешном завершении записи Платеж со статусом `успешен` автоматически:

  - создавать связанную запись Чек
  - заполнять `идПлатежа`
  - устанавливать `сумма`, равную сумме платежа
  - устанавливать `датаВремяЧека` текущим системным значением
  - задавать `статусФискализации` = `ожидание`

- Чек.Изменить.СД.статусФискализации

  Система должна для записей Чек со `статусФискализации` = `ожидание`:

  - формировать запрос к фискальному регистратору/ОФД с необходимыми реквизитами чека
  - по результатам присваивать атрибуту `номерФискальный` значение, полученное от внешней системы
  - при необходимости обновлять `датаВремяЧека`
  - изменять `статусФискализации` на значения из фиксированного набора

- Чек.ПовторноОтправить.СД

  Система должна для записей Чек со `статусФискализации` = `ошибка`:

  - обеспечивать возможность раз в сутки повторного формирования и отправки данных чека во внешнюю фискальную систему
  - по результатам повторной попытки обновлять:
    - номерФискальный
    - датаВремяЧека
    - статусФискализации

### Проверка полноты набора ФТ

- Основные потоки UC-10.6 и смежных сценариев оплаты покрыты:
  - просмотр списка чеков
  - отчётность по чекам
  - автоматическое создание чека
  - фискализация и повторная отправка
- Потенциально отсутствующие ФТ (рекомендации к расширению):
  - ФТ по просмотру детальной карточки чека
  - ФТ по скачиванию или отправке чека клиенту
  - ФТ по повторной выдаче чека по запросу клиента
