## Функциональные требования: объект Шаблон уведомления

### Входные UC

- UC-6.1 «Создать шаблон уведомления»
- UC-6.2 «Просмотреть шаблон уведомления»
- UC-6.3 «Изменить шаблон уведомления»
- UC-6.4 «Удалить шаблон уведомления»
- UC-6.5 «Просмотреть список шаблонов уведомлений»

### Кандидаты ФТ (иерархия Роль.Объект.Действие)

## Управляющий

- ШаблонУведомления.Создать

  Система должна позволять пользователю с ролью Управляющий создавать объект Шаблон уведомления со следующими атрибутами:

  - идентификатор
  - код
  - наименование
  - тип
  - тема
  - тело

- ШаблонУведомления.Просмотреть

  Система должна позволять пользователю с ролью Управляющий просматривать объект Шаблон уведомления со следующими атрибутами:

  - идентификатор
  - код
  - наименование
  - тип
  - тема
  - тело

- ШаблонУведомления.Изменить

  Система должна позволять пользователю с ролью Управляющий изменять объект Шаблон уведомления со следующими атрибутами:

  - наименование
  - тип
  - тема
  - тело

- ШаблонУведомления.Удалить

  Система должна позволять пользователю с ролью Управляющий помечать объект Шаблон уведомления как удалённый либо недоступный для использования в новых уведомлениях с изменением следующих атрибутов:

  - признакАктуальности = false

- ШаблонУведомления.ПросмотретьСписок

  Система должна позволять пользователю с ролью Управляющий просматривать список объектов Шаблон уведомления с отображением для каждого шаблона следующих атрибутов:

  - идентификатор
  - код
  - наименование
  - тип

## Системы

- ШаблонУведомления.ПроверитьПлейсхолдеры.СД.тема

  Система должна проверять `[ШаблонУведомления]:тема` по следующему алгоритму:

  - при сохранении шаблона система должна анализировать тему на наличие плейсхолдеров в согласованном формате, например `{{имяКлиента}}` и `{{номерБронирования}}`
  - если в теме обнаружены плейсхолдеры, отсутствующие в перечне допустимых для выбранного типа шаблона, система должна отклонять сохранение шаблона с указанием списка некорректных плейсхолдеров и не изменять сохранённое значение `[ШаблонУведомления]:тема`

- ШаблонУведомления.ПроверитьПлейсхолдеры.СД.тело

  Система должна проверять `[ШаблонУведомления]:тело` по следующему алгоритму:

  - при сохранении шаблона система должна анализировать тело на наличие плейсхолдеров в согласованном формате, например `{{имяКлиента}}` и `{{номерБронирования}}`
  - если в теле обнаружены плейсхолдеры, отсутствующие в перечне допустимых для выбранного типа шаблона, система должна отклонять сохранение шаблона с указанием списка некорректных плейсхолдеров и не изменять сохранённое значение `[ШаблонУведомления]:тело`
  - если все плейсхолдеры в теме и теле корректны, система должна разрешать сохранение шаблона без дополнительных изменений значений `[ШаблонУведомления]:тема` и `[ШаблонУведомления]:тело`

- ШаблонУведомления.Создать.СД.идентификатор

  Система должна изменять `[ШаблонУведомления]:идентификатор` по следующему алгоритму:

  - `[ШаблонУведомления]:идентификатор` = max(`[ШаблонУведомления]:идентификатор`) + 1
