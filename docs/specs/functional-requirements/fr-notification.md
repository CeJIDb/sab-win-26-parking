## КлиентФЛ

- Уведомление.Просмотреть

  Система должна позволять пользователю с ролью КлиентФЛ просматривать принадлежащий ему объект Уведомление со следующими атрибутами:

  - идентификатор
  - `[ШаблонУведомления]:код`
  - канал
  - статусДоставки

  - идентификатор
  - `[ШаблонУведомления]:код`
  - канал
  - статусДоставки

## Системы

- Уведомление.Создать.СД

  Система должна изменять Уведомление со следующими атрибутами:

  - идентификатор
  - канал

- Уведомление.Создать.СД.канал

  Система должна изменять `[Уведомление]:канал` по следующему алгоритму:

  - если для клиента в сущности Настройки уведомлений указано несколько каналов, то система должна выбирать канал в следующем приоритетном порядке: «личный кабинет», затем «e-mail», затем «SMS»
  - если для клиента доступен только один канал, то система должна устанавливать `[Уведомление]:канал` равным этому каналу
  - если для клиента не включён ни один канал для соответствующего типа уведомления, система не должна создавать объект Уведомление и должна зафиксировать факт невозможности отправки в журнале аудита

- Уведомление.Создать.СД.идентификатор

  Система должна изменять `[Уведомление]:идентификатор` по следующему алгоритму:

  - `[Уведомление]:идентификатор` = max(`[Уведомление]:идентификатор`) + 1

- Уведомление.Изменить.СД

  Система должна изменять Уведомление со следующими атрибутами:

  - статусДоставки

- Уведомление.Изменить.СД.статусДоставки

  Система должна изменять `[Уведомление]:статусДоставки` по следующему алгоритму:

  - при получении успешного ответа от смежной системы доставки для конкретного уведомления `[Уведомление]:статусДоставки` = «успешно»
  - при получении сообщения об ошибке доставки для конкретного уведомления `[Уведомление]:статусДоставки` = «ошибка»
