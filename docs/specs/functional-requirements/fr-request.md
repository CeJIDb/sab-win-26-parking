## Функциональные требования: объект Обращение

### Входные UC

- UC-4.1 «Создать обращение»
- UC-4.2 «Просмотреть обращение»
- UC-4.3 «Просмотреть список обращений»
- UC-4.4 «Просмотреть обращение клиента»
- UC-4.5 «Ответить на обращение клиента»
- UC-4.6 «Просмотреть список обращений клиентов»

### Кандидаты ФТ (иерархия Роль.Объект.Действие)

### Реестр идентификаторов требований (FR-*)

Обоснование: для трассируемости и верифицируемости каждое требование из списка кандидатов получает уникальный идентификатор формата `FR-*`.

- **FR-REQUEST-001** `Обращение.Создать`
- **FR-REQUEST-002** `Обращение.Просмотреть`
- **FR-REQUEST-003** `Обращение.ПросмотретьСписок`
- **FR-REQUEST-004** `Обращение.Ответить`
- **FR-REQUEST-005** `Обращение.ОбновитьСтатус.СД`
- **FR-REQUEST-006** `Обращение.ОбновитьСтатус.СД.статус`
- **FR-REQUEST-007** `Обращение.Создать.СД.идентификатор`

## КлиентФЛ

- Обращение.Создать

  Система должна позволять пользователю с ролью КлиентФЛ создавать объект Обращение со следующими атрибутами:

  - идентификатор
  - клиентИд
  - тип
  - канал
  - тема
  - описание
  - типСвязанногоОбъекта
  - связанныйОбъектИд
  - статус

- Обращение.Просмотреть

  Система должна позволять пользователю с ролью КлиентФЛ просматривать принадлежащий ему объект Обращение со следующими атрибутами:

  - идентификатор
  - `[Сотрудник]:логин`
  - тип
  - канал
  - тема
  - описание
  - типСвязанногоОбъекта
  - связанныйОбъектИд
  - статус

- Обращение.ПросмотретьСписок

  Система должна позволять пользователю с ролью КлиентФЛ просматривать список принадлежащих ему объектов Обращение с отображением для каждого обращения следующих атрибутов:

  - идентификатор
  - тип
  - тема
  - типСвязанногоОбъекта
  - связанныйОбъектИд
  - статус

## Системы

- Обращение.ОбновитьСтатус.СД

  Система должна изменять Обращение со следующими атрибутами:

  - статус

- Обращение.ОбновитьСтатус.СД.статус

  Система должна изменять `[Обращение]:статус` по следующему алгоритму:

  - если по обращению создан ответ сотрудника и `[Обращение]:статус` ∈ {«новое», «в работе»}, то `[Обращение]:статус` = «ожидает клиента»
  - если клиент подтвердил решение или явно закрыл обращение, то `[Обращение]:статус` = «закрыто»
  - если с момента перевода статуса в «ожидает клиента» прошло более 14 календарных дней без активности клиента, то `[Обращение]:статус` = «решено»

- Обращение.Создать.СД.идентификатор

  Система должна изменять `[Обращение]:идентификатор` по следующему алгоритму:

  - `[Обращение]:идентификатор` = max(`[Обращение]:идентификатор`) + 1
