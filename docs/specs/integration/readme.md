# Индекс интеграционных требований

Раздел для требований класса `INT-*`, которые описывают, какие данные, статусы и взаимодействия система должна поддерживать при обмене с внешними и смежными системами.

## Оглавление

- [Ключевые документы](#ключевые-документы)
- [Граница раздела](#граница-раздела)
- [Связанные документы](#связанные-документы)

## Ключевые документы

- [Требования к интеграции](integration-requirements.md) — каноничный перечень требований класса `INT-*`, включая платежи, ОФД, СКУД, IdP, уведомления, ЭДО и табло.

## Граница раздела

В `docs/specs/integration/` хранятся именно требования:

- что система должна принимать и передавать;
- какие атрибуты обязательны;
- какие статусы и результаты должны поддерживаться;
- какие интеграционные гарантии ожидаются на уровне требований.

В `docs/architecture/integration/` хранятся архитектурные и проектные материалы:

- sequence-диаграммы;
- DFD и message flow;
- API-контракты и provider-specific маппинги;
- решения по ретраям, идемпотентности, журналированию и обработке ошибок.
