# Контекстная диаграмма MVP

Текстовое представление по результатам чтения тайлов `контекстная_диаграмма_mvp_tiles/tile_r*_c*.png` (исходник: `Контекстная диаграмма MVP.jpg`).

**Терминология (по протоколам №4, №5):** ГРЗ — государственный регистрационный знак (идентифицирует ТС); ПМ — парковочное место; ТС — транспортное средство. Тариф, бронирование, договор — по глоссарию протокола №4 (User Story Map).

---

## Центральная система

**Цифровая платформа парковок** — ядро MVP в границах контекстной диаграммы.

С платформой взаимодействуют внешние акторы, платежные и фискальные системы, СКУД, сервис уведомлений, информационные табло и дисплеи на въезде/выезде.

---

## Внешние акторы и обмен данными

### 1. ЮЛ (юридическое лицо)

*По протоколу №4: заключение договоров с ЮЛ в MVP — вне системы (ручной режим). В платформе: автоматическое распознавание и пропуск; управление списком ТС (добавление, изменение, удаление); делегации (доп. места на период).*

**ЮЛ → Система (в систему):**

- Данные представителя: номер телефона, ФИО, e-mail
- ГРЗ (идентификация ТС)
- ПМ (параметры мест по договору)

**Система → ЮЛ (из системы):**

- Данные представителя: номер телефона, ФИО, e-mail
- ГРЗ, управление списком ТС
- Договор (файл + параметры)
- Назначенные ПМ или диапазон по договору
- Парковочная сессия
- Уведомления

---

### 2. ФЛ (договор) — физическое лицо с договором

**ФЛ (договор) → Система (в систему):**

- Данные клиента: номер телефона, ФИО, e-mail
- Данные для заключения договора: дата рождения, документ удостоверяющий личность, адрес регистрации, скан/фото паспорта
- ГРЗ
- Параметры договора: срок, тариф, количество мест
- Оплата: способ оплаты, согласие на автосписание
- Юридические согласия: оферта, обработка ПДн

**Система → ФЛ (договор) (из системы):**

- Профиль клиента: номер телефона, ФИО, e-mail
- Договор (файл + параметры)
- ГРЗ
- Оплата: способ оплаты, статус автосписания, сумма к оплате, статус оплаты, квитанция об оплате
- Парковочная сессия
- Уведомления

---

### 3. ФЛ (почасовая) — физическое лицо, почасовая парковка

**ФЛ (почасовая) → Система (в систему):**

- Данные клиента: номер телефона, ФИО, e-mail
- ГРЗ
- Оплата: способ оплаты, согласие на автосписание
- Юридические согласия: оферта, обработка ПДн

**Система → ФЛ (почасовая) (из системы):**

- Профиль клиента: номер телефона, ФИО, e-mail
- ГРЗ
- Оплата: способ оплаты, статус автосписания, сумма к оплате, статус оплаты, квитанция об оплате
- Тариф
- Парковочная сессия
- Уведомления

---

### 4. ФЛ (без ЛК) — физическое лицо без личного кабинета

*По протоколу №5: заказчик настаивает на минимальной идентификации (телефон + имя). На въезде — явное согласие с тарифом и правилами (терминал/кнопка). Постоплата на выезде по фактическому времени сессии. По ручному уточнению (Контекстная диаграмма MVP.txt): напрямую с системой не взаимодействует; только через вторые касания (СКУД, табло, персонал).*

**Напрямую с платформой не взаимодействует** — только через СКУД, информационное табло КПП, дисплеи на въезде/выезде, оператора (терминал, ручной ввод).

**Система → ФЛ (без ЛК)** (косвенно, через табло/дисплей): статус КПП, инструкция, количество свободных мест.

---

### 5. Оператор (Охранник)

**Оператор → Система (в систему):**

- ГРЗ (ручной ввод)
- Решение по доступу
- Данные клиента (поиск профиля)
- Профиль ФЛ
- Парковочная сессия (ручное создание)

**Система → Оператор (из системы):**

- Результат проверки доступа
- Профиль ЮЛ, Профиль ФЛ
- ГРЗ
- Статус оплаты
- Парковочная сессия
- Инструкции
- Системные уведомления (АПС)
- Обращения, данные договора *(модуль обращений — возможно не в MVP; протокол №5)*

---

### 6. Администратор (Управляющий / Владелец)

**Администратор → Система (в систему):**

- Профиль ЮЛ, Профиль ФЛ
- Договор ЮЛ, Договор ФЛ
- ГРЗ
- Право доступа, Оплата, Тариф
- Правила пользования парковкой
- Шаблон договора ФЛ, Шаблон договора ЮЛ
- Оферта
- ПМ, Сектор
- Профиль Охранника (управление сотрудниками)

**Система → Администратор (из системы):**

- Профиль ЮЛ, Профиль ФЛ
- Договор ЮЛ, Договор ФЛ
- ГРЗ, Право доступа, Оплата, Тариф
- Правила пользования парковкой, Шаблон договора ФЛ/ЮЛ, Оферта
- ПМ, Сектор, Охранник
- Парковочная сессия
- **Доп. права владельца:** Управляющий, Аналитика *(на релизе MVP аналитика в специальном виде не в приоритете; протокол №4)*
- Обращения, данные договора; тарифы, правила, контакты, страницы *(при наличии модуля обращений)*

## Внешние системы

### 7. Платежная система (онлайн)

*По протоколу №5: основной онлайн-сценарий — оплата через ЛК и автосписание; вопрос по MVP — только ЛК или ЛК + QR.*

**Платежная система → Система (в систему):** Статус платежа, id платежа, Квитанция об оплате, Статус возврата  
**Система → Платежная система (из системы):** Сумма к оплате, id платежа, Сумма к возврату (при возврате)

---

### 8. Платежный терминал (офлайн)

*Резерв у охранника на КПП; опция — терминал у шлагбаума на выезде (протокол №5).*

**Платежный терминал → Система (в систему):** Статус платежа, id платежа, Квитанция об оплате  
**Система → Платежный терминал (из системы):** Сумма к оплате, id платежа

---

### 9. ОФД (оператор фискальных данных)

**Система → ОФД (из системы в ОФД):** Чек к фискализации, id платежа  
**ОФД → Система (в систему от ОФД):** Фискальный чек, Статус фискализации

---

### 10. Сервис уведомлений

**Система → Сервис уведомлений (из системы в сервис):**

- ОТП-код (код подтверждения)
- Уведомление
- Канал уведомления
- Адресат: номер телефона, e-mail

**Сервис уведомлений → Система (в систему от сервиса):** Статус отправки  
*(Уведомления доставляются адресатам через сервис; к платформе возвращается только статус.)*

---

### 11. СКУД (система контроля и управления доступом)

**СКУД → Система (в систему от СКУД):**

- ГРЗ (распознанный номер)
- Вероятность распознавания
- Дата и время распознавания
- Полоса (въезд/выезд)
- Номер КПП
- ID кадра

**Система → СКУД (из системы в СКУД):**

- Право доступа (разрешен / запрещен)

---

### 12. Геосервисы и Сайт-визитка

**Система → Геосервисы / Сайт-визитка (из системы):** информация о работе, правилах, тарифах парковки; количество свободных мест *(в т.ч. для размещения на геосервисах)*  
*(Обратного потока от сайта/гео к платформе в контексте диаграммы нет — только передача данных от платформы.)*

---

## Оборудование и интерфейсы на территории

### 13. Информационное табло КПП (автономное КПП)

**Система → Информационное табло КПП (из системы):**

- Количество свободных мест
- Статус работы КПП
- Инструкции водителю
- Предупреждения и правила
- Аварийные сообщения

*(Обратного потока от табло к платформе нет. К водителю/ФЛ без ЛК — визуальное отображение: статус КПП, инструкция, количество свободных мест.)*

---

### 14. Дисплей стойки на въезде

**Система → Дисплей стойки на въезде (из системы):** ГРЗ, Право доступа, Инструкции водителю, Тариф, Назначенные ПМ/сектор  
*(К водителю — визуальное отображение.)*

---

### 15. Дисплей стойки на выезде

**Система → Дисплей стойки на выезде (из системы):** ГРЗ, Право доступа, Тариф, Сумма к оплате, Статус платежа, Инструкции водителю  
*(К водителю — визуальное отображение.)*

## Сводка: данные на границе с платформой

*Направления потоков приведены в соответствие с ручным описанием (Контекстная диаграмма MVP.txt).*

**В систему (к платформе):** ГРЗ и метаданные от СКУД; личные данные и профили от клиентов (ЮЛ, ФЛ договор, ФЛ почасовая) и администратора; данные для договора, параметры договора, согласия; от оператора — ГРЗ (ручной ввод), решение по доступу, профиль ФЛ, парковочная сессия (ручное создание); от платёжных систем и терминала — статус платежа, id платежа, квитанция; от ОФД — фискальный чек, статус фискализации; от сервиса уведомлений — статус отправки.

**Из системы (от платформы):** Профили, договоры, ГРЗ, назначенные ПМ, парковочная сессия, уведомления (к клиентам); к оператору — результат проверки доступа, профили, ГРЗ, статус оплаты, инструкции, системные уведомления; к администратору — профили, договоры, тарифы, оферта, ПМ, сектор, охранник, парковочная сессия, аналитика; в СКУД — право доступа; в платёжные систему/терминал — сумма к оплате, id платежа, сумма к возврату; в ОФД — чек к фискализации, id платежа; в сервис уведомлений — OTP, уведомление, канал, адресат; в геосервисы/сайт-визитку — информация, тарифы, количество свободных мест; на табло и дисплеи — данные для отображения водителю.

## Соответствие протоколам и артефактам

- **Протокол №4:** Центральная система = Consumer App (PWA) + админ-панель + Business Site; терминология тариф/бронирование/договор; ЮЛ — договор вне MVP, доступ и список ТС в MVP; аналитика на релизе не в приоритете; контактные данные от всех клиентов (включая краткосрочных) обязательны.
- **Протокол №5:** ФЛ без ЛК и ФЛ почасовая — постоплата на выезде; ФЛ (договор) — оплата договора онлайн; бесконтактный въезд-выезд по ГРЗ; парковочная сессия создаётся на въезде, завершается на выезде; дисплеи на въезде/выезде — по перечню в Event Storming TO-BE; минимальная идентификация для ФЛ без ЛК (телефон + имя); явное согласие с офертой/правилами.
- **User Story Map:** Акторы (клиент, владелец, управляющий, охранник), блоки регистрации, авторизации, ТС, бронирования, договора, оплаты, доступа, мощностей — отражены в акторах и потоках контекстной диаграммы.
- **Impact Map:** Личный кабинет клиента, админ-панель, СКУД, платежный модуль, интерфейс КПП для охранника, геосервисы, сайт-визитка — соответствуют внешним системам и акторам на диаграмме.
