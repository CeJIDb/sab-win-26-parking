## Назначение

Текстовое древовидное представление DFD L1 в физической раскладке: 10 внешних систем, 20 модулей бэкенда и 1 СУБД PostgreSQL вместо 11 логических хранилищ. Подходит для чтения без рендеринга Mermaid и для быстрой навигации по соседям каждого узла.

Акторы (ФЛ, ЮЛ, охранник КПП, управляющий, владелец) на L1 не показаны — см. [DFD L0](context-diagram.md). Управляющие потоки и протоколы транспорта не отражены — это уровень L2.

## Условные обозначения

- `→` — исходящий поток данных (узел отправляет данные соседу).
- `←` — входящий поток данных (узел получает данные от соседа).
- `↔` — двусторонний обмен (запрос/ответ или прямой и обратный поток).
- Префиксы: `E` — внешняя система, `P` — процесс бэкенда, `DB` — единая СУБД PostgreSQL.

## Сводка по связям с БД

| Группа                    | Модули                                                         |
| ------------------------- | -------------------------------------------------------------- |
| Пишут и читают БД         | P1, P2, P3, P4, P5, P6, P8, P9, P10, P11, P12, P13, P14, P15   |
| Только читают БД          | P16 (тариф для расчета стоимости)                              |
| Не работают с БД напрямую | P7 (SSO-делегат), P17, P18, P19, P20 (адаптеры внешних систем) |
