## UC-12.9 «Завершить бронирование автоматически на выезде»

### 1. Область действия / контекст

Автоматическое завершение бронирования при выезде ТС с парковки, если ПС была связана с конкретным бронированием (ручным или автоматическим).

### 2. Акторы

- **ИС парковки** – завершает связанное бронирование.
- **Клиент ФЛ / Клиент ЮЛ** – косвенный участник (не взаимодействует напрямую).

### 3. Цель

Корректно завершить бронирование, когда ТС фактически покидает парковку, чтобы:

- освободить ресурс (место/квоту);
- не блокировать дальнейшие бронирования;
- корректно отразить использование услуги.

### 4. Предусловия

- UC-12.8 «Завершить ПС» инициировал проверку связанного Бронирования (альтернативный поток 7а).
- ПС связана с активным краткосрочным или автоматическим Бронированием, созданным в UC‑12.2 или UC‑7.1/7.6.

### 5. Триггер

- Система определяет по данным из UC‑12.9, что тип связанного Бронирования не «долгосрочное» и Бронирование подлежит завершению при выезде ТС.

### 6. Постусловия / результат

- связанное краткосрочное/автоматическое Бронирование переведено в статус «Завершено»;
- изменен атрибут «Зарезервировано» = 0 связанного ПМ.

### 7. Основной поток

1. Система изменяет Бронирование:
   - статус = «Завершено».
2. Система изменяет связанное с Бронированием ПМ:
   - «Зарезервировано» = 0.
3. Система завершает выполнение сценария.
