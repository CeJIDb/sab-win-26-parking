## Назначение

Реестр Use Case фиксирует каноничный перечень вариантов использования проекта, их группировку и основных участников.

## Контекст и источник

- Этап проекта: Этап 2. Моделирование взаимодействия и разработка пользовательских требований
- Тип артефакта: Реестр use case
- Источник: UML Use Case Diagram, User Story Map, проектное моделирование команды
- Статус: каноничный реестр UC проекта

## Диаграмма

В текущей версии отдельное изображение не приложено; визуальным дополнением выступает [Диаграмма вариантов использования](use-case-diagram.md).

## Текстовое описание

Документ собирает в одном месте все варианты использования проекта, связывая их с группами функций и основными участниками. Реестр нужен как точка навигации между высокоуровневой диаграммой вариантов использования, детальными UC и функциональными требованиями.

## Ключевые элементы

- Участники сценариев
- Группы функциональности
- Код UC
- Название UC

## Логика артефакта

Реестр организован таблично и позволяет быстро увидеть покрытие системы по ролям и функциональным блокам. Он нужен не для описания шагов сценария, а для инвентаризации состава будущей системы и связи между артефактами требований.

## Выводы и решения

- Реестр показывает полный функциональный охват проекта по ролям и доменным блокам.
- Он служит основой для детализации use case, FR и навигации по проектной документации.
- Табличный формат делает видимыми пробелы и перекосы по функциональным группам.

## Ограничения и открытые вопросы

- Реестр не заменяет детальные сценарии и не описывает потоки шагов, правил и исключений.
- При изменении состава системы и MVP его нужно синхронизировать с диаграммой use case и детальными UC.
