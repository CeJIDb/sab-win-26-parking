## UC-12.8 «Завершить ПС»

### 1. Область действия / контекст

Завершение парковочной сессии (ПС) после выезда ТС с парковки с расчетом итоговой стоимости и формированием требований к оплате или фиксацией использования в рамках уже оплаченных услуг.

### 2. Акторы

- **ИС парковки** – завершает ПС и выполняет расчеты.
- **Клиент ФЛ / Клиент ЮЛ** – косвенный участник (результат – начисления/оплаты).

### 3. Цель

Корректно завершить ПС, зафиксировав ее завершение и освободив связанное парковочное место для дальнейшего использования.

### 4. Предусловия

- UC-12.6 «Пройти автоматическую идентификацию на выезде» выполнен.
- Для ТС найдена активная ПС (UC-12.4).

### 5. Триггер

- ТС выезжает с парковки и выходит из зоны действия камер СКУД.

### 6. Постусловия / результат

- ПС переведена в статус «Завершена».
- изменен атрибут «Занято» = 0 связанного ПМ;
- если ПС связана с краткосрочным/автоматическим Бронированием, инициировано завершение Бронирования (UC‑12.7).

### 7. Основной поток

1. Система принимает сообщение от СКУД:
   - тип события = «выезд из зоны»;
   - ГРЗ;
   - дату и время события;
   - идентификатор КПП;
   - идентификатор полосы.
2. Система изменяет связанную ПС с Бронированием из UC‑12.6:
   - статус = «Завершена».
3. Система изменяет ПМ из Бронирования:
   - признак фактической занятости («Занято») = 0.
4. Система пересчитывает количество свободных мест.
5. Система передает количество свободных мест на Информационное табло.
6. Система актуализирует информацию о свободных местах:
   - на Сайте‑визитке (Главная);
   - в ЛК клиента (раздел «Парковка»);
   - в Интерфейсе КПП (Панель КПП);
   - в ЛК управляющего и владельца (Главная).
7. Система читает тип связанного Бронирования.
8. Система завершает выполнение сценария.

### 8. Альтернативные и исключительные потоки

**7а. Тип Бронирования ≠ «долгосрочное»**

7а1. Система завершает выполнение сценария и передает управление в UC‑12.7 «Завершить бронирование автоматически на выезде».
