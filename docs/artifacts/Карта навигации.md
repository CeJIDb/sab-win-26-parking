# Карта навигации

Навигационная карта (sitemap) структуры сайта и личных кабинетов. Составлена по реальному UI (каталог `ui/`, шаблоны навигации и страницы) с учётом артефактов проекта; расхождения отражены в разделе «Отчёт по составлению карты навигации».

---

## Условные обозначения

| Цвет узла | Назначение |
|-----------|------------|
| Фиолетовый | Точка входа / главная (сайт‑визитка) |
| Зелёный | Публичные разделы и личный кабинет клиента |
| Оранжевый | Личный кабинет администратора и рабочие интерфейсы персонала |
| Розовый | Внешняя система (интеграция / аналитика) |

---

## 1. Сайт‑визитка (точка входа)

*Фиолетовый узел — публичный сайт и стартовая точка навигации.*

**Точка входа (index.html):**

- **Сайт парковки** → `site/index.html`
- **Личный кабинет клиента** → `site/login.html`
- **Вход для сотрудников** → `site/staff-login.html`

**Навигация сайта (nav-site):**

- **Главная** — `site/index.html` (описание парковки, компактный блок карты по секторам на главной)
- **Тарифы и условия** — `site/tariffs.html`
- **Правила парковки** — `site/rules.html`
- **Контакты** — `site/contacts.html`

**В шапке сайта:**

- **Регистрация** — `site/register.html`
- **Вход** — `site/login.html`

**В футере сайта:**

- **Вход для сотрудников** — `site/staff-login.html`

**Дополнительно:**

- Отдельная страница `site/parking-map.html` есть в UI, но в основном меню сайта ссылки на неё нет; карта по секторам выведена на главной странице (компактный блок).

## 2. Личный кабинет клиента

*Зелёная ветка — клиентская зона после успешной авторизации (вход через `site/login.html`).*

**Навигация ЛК клиента (nav-lk.njk):**

- **Парковка** — `client/dashboard.html` (главная: быстрые действия, карта, тарифы; из блоков доступны: История парковочных сессий → `sessions-history.html`, Бронирования → `bookings.html`, Договоры → `contracts.html`, создание договора → `contracts-create.html`)
- **Оплаты** — `client/payment-settings.html`
- **Профиль** — `client/profile.html`

Пунктов «Обращения», «Бронирование», «Договоры», «История сессий» в основном меню нет — они доступны с дашборда (Парковка). Страницы без пункта в меню (переход с дашборда): `client/bookings.html`, `client/contracts.html`, `client/contracts-create.html`, `client/sessions-history.html`. **Обращения** — отдельной страницы `appeals.html` в UI нет; доступны **из профиля** (раздел «Поддержка»): кнопка «Мои обращения» открывает модалку `#profile-appeals-modal`, «Написать в поддержку» — `#appeal-create-modal`. В уведомлениях ссылка на ответ по обращению ведёт на `profile.html#profile-appeals-modal`.

**Переходы из профиля (client/profile.html):**

- **Данные и настройки:** «Изменить данные» → `#profile-edit-modal`, «Изменить пароль» → `#password-change-modal`, «Настройки» → `#profile-settings-modal`, «Список ТС» → `#profile-vehicles-modal` (и связанные модалки добавления/редактирования/удаления ТС).
- **Данные организации (ЮЛ):** «Добавить данные организации» → `#profile-organization-add-modal`, «Изменить данные организации» → `#profile-organization-edit-modal`.
- **Поддержка:** «Мои обращения» → `#profile-appeals-modal`, «Написать в поддержку» → `#appeal-create-modal`, «Позвонить в поддержку» (tel:), «FAQ» → `../site/index.html#faq`.
- **Выход** → `../site/login.html`.

**В шапке ЛК клиента:**

- Иконка **Уведомления** → `client/notifications.html`
- Имя пользователя → `client/profile.html`

**Дополнительные страницы (переходы с дашборда/разделов):**

- `client/contracts-create.html` — создание договора
- `client/contract-sign.html` — подписание договора кодом из SMS (после формирования договора); далее переход к оплате
- `client/payment-external.html` — внешняя оплата (переход из активной сессии / договора / раздела «Бронирование» / после подписания договора)

**Особенности навигации и адаптации ЛК клиента:**

- навигация ЛК (`nav-lk`) сворачивается в бургер‑меню на мобильных устройствах; состояние меню управляется атрибутом `data-nav` на `body` и кнопкой‑бургером в шапке;
- в футере ЛК есть переключатель версии интерфейса «для компьютера / для телефона», выбор сохраняется в `localStorage` (`lkView`) и отражается в атрибуте `data-view` на `body`; это влияет на представление интерфейса, но не создаёт отдельных страниц или узлов навигации.

## 3. Личный кабинет администратора

*Оранжевая ветка — административный интерфейс (вход через `site/staff-login.html`, выбор роли администратора).*

**Вход:** только через `site/staff-login.html` с выбором роли администратора; после выбора роли — переход на главную админки (`admin/index.html`).

**Навигация ЛК администратора (nav-role при role=admin):**

- **Главная** — `admin/index.html`
- **Парковка** — по умолчанию `admin/sessions.html` (Парковочные сессии); при переходе в раздел отображается вспомогательная навигационная панель (подразделы):
  - **Парковочные сессии** — `admin/sessions.html`
  - **Секторы и ПМ** — `admin/sectors.html`
  - **Тарифы** — `admin/tariffs.html`
  - **Уведомления** — `admin/notifications-manage.html` (управление уведомлениями)
  - **Профили сотрудников** — `admin/admins.html` (профили админов)
  - **Аналитика** — переход на внешний ресурс *(розовый узел — внешняя система/интеграция)*, без внутренней страницы; кнопка выровнена по правому краю вспомогательной панели
- **Клиенты** — по умолчанию `admin/debts.html`; при переходе в раздел отображается вспомогательная навигационная панель (подразделы):
  - **Задолженности** — `admin/debts.html`
  - **Бронирования** — `admin/bookings.html`
  - **Договоры** — `admin/contracts.html`
  - **Профили** — `admin/clients.html` (профили клиентов)
- **Обращения** — `admin/requests.html`
- **КПП** — по умолчанию `admin/kpp-panel.html`; при переходе в раздел отображается вспомогательная навигационная панель (подразделы):
  - **Панель** — `admin/kpp-panel.html`
  - **Журнал въезда‑выезда** — `admin/kpp-log.html`
- **Профиль** — `admin/profile.html`

Вспомогательные панели подменю задаются партиалами: **Парковка** — `nav-parking.njk`, **Клиенты** — `nav-clients.njk`, **КПП** — `nav-kpp.njk`.

**В шапке ЛК администратора:**

- Иконка **Уведомления** → `admin/notifications.html`
- Имя/профиль администратора → `admin/profile.html`

**Реестр договоров (admin/contracts.html):** в таблице в колонке «Действия» три кнопки — **Просмотр**, **Изменить**, **Расторгнуть**; они открывают соответственно модалки просмотра договора (`#admin-contract-modal`), редактирования (`#admin-contract-edit-modal`), расторжения (`#admin-contract-terminate-modal`). Блок расторжения вынесен из модалки просмотра в отдельную модалку.

**Страница «Парковочные сессии» и карточка клиента:**

- `admin/sessions.html` — парковочные сессии (первый пункт вспомогательной навигации «Парковка», по умолчанию при входе в раздел Парковка).
- `admin/client-summary.html` — карточка клиента в админке; доступна при переходе из панели КПП или журнала КПП в контексте админки (`from_admin`). С карточки клиента и из модалок — переход на `admin/client-sessions.html` (сессии по ГРЗ/клиенту).

**Дополнительные страницы и внутренние представления:**

- `admin/contracts-create.html` — создание договора
- `admin/client-sessions.html` — сессии клиента (переход с карточки клиента `client-summary.html` и из модалок)
- `admin/debts.html` — страница задолженностей: сводка по задолженностям (в т.ч. блок «Просроченные оплаты» с кнопкой «Показать» → якорь на реестр `#debts-register`), реестр с фильтрами; действия «Договор» и «Карточка клиента» открывают модальные окна без перехода на отдельные страницы (`#admin-contract-modal`, `#admin-client-fl-modal`, `#admin-client-yl-modal`). В карточке клиента (ФЛ) доступно редактирование ТС: добавить / изменить / удалить через модальные окна (`#admin-client-vehicle-add-modal`, `#admin-client-vehicle-edit-modal`, `#admin-client-vehicle-delete-modal`).

## 4. Рабочий интерфейс охранника КПП

*Оранжевая ветка — отдельный контур; вход через `site/staff-login.html` с выбором роли охранника.*

**Навигация (nav-role.njk при role=operator):**

- **КПП — панель** — `guard/index.html`
- **Журнал въезда‑выезда** — `guard/log.html`
- **Профиль** — `guard/profile.html`

**В шапке интерфейса охранника:**

- Иконка **Уведомления** → `guard/notifications.html`
- Имя/профиль сотрудника → `guard/profile.html` (пункт «Профиль» выведен также в основном меню nav-role.njk)

**Переходы с панели и журнала:**

- Карточка клиента/ТС → `guard/client-summary.html`
- С карточки клиента → `guard/plate-history.html` (история по номеру)

**Выход:** в разметке layout guard футера с ссылкой «Выход» нет; возврат к `site/staff-login.html` оформляется через профиль или иной контекст выхода.

## Отчёт по составлению карты навигации

Карта приведена в соответствие с реальной структурой UI. **Источники:** шаблоны навигации `nav-lk.njk`, `nav-role.njk`, `nav-parking.njk`, `nav-clients.njk`, `nav-kpp.njk`; layout'ы `admin.njk`, `guard.njk`, `lk.njk`; страницы в `ui/admin/`, `ui/client/`, `ui/guard/`; партиалы модалок (договоры, карточка клиента, ТС). Дополнительно: предыдущая версия артефакта, User Story Map, Концептуальная модель (docs/artifacts/).

**Порядок вызова модалок** в дереве (раздел 5) проверен по ссылкам `href="#..."` в соответствующих партиалах: каждая дочерняя модалка открывается из той родительской модалки или страницы, под которой она указана. Отличие: в ЛК клиента «Удалить ТС» открывается из Списка ТС (рядом с «Редактировать»); в ЛК администратора «Удалить ТС» открывается из модалки «Редактировать ТС».

**Актуализация (текущее состояние UI):**

- **ЛК клиента:** в меню (nav-lk.njk) только три пункта — Парковка (dashboard), Оплаты, Профиль; Бронирования, Договоры, История сессий доступны с дашборда. Обращения доступны из профиля (раздел «Поддержка»): модалки «Мои обращения» (#profile-appeals-modal) и «Написать в поддержку» (#appeal-create-modal); отдельной страницы `appeals.html` нет. В разделе 2 добавлен блок «Переходы из профиля» — данные/настройки/ТС, данные организации, поддержка (обращения, FAQ), выход.
- **ЛК администратора:** добавлены страницы `admin/sessions.html` (Парковочные сессии — первый пункт подменю «Парковка») и `admin/client-summary.html` (карточка клиента; вход из КПП в контексте from_admin); переходы: с карточки клиента и из модалок → `admin/client-sessions.html`. Layout admin.njk: футер со ссылкой «Выход» → site/staff-login.html, переключатель версии интерфейса (adminView в localStorage).
- **Охранник КПП:** в nav-role.njk при role=operator пункт «Профиль» выведен в основном меню; формулировка в карте исправлена. В layout guard.njk футера с «Выход» в разметке нет.
