# Карта навигации

Навигационная карта (sitemap) структуры сайта и личных кабинетов. Текстовое представление по результатам чтения тайлов `карта_навигации_tiles/tile_r*_c*.png` (исходник: `Карта навигации.jpg`).

---

## Условные обозначения

| Цвет узла | Назначение |
|-----------|------------|
| Фиолетовый | Точка входа / главная (сайт-визитка) |
| Зелёный | Публичные разделы и личный кабинет клиента |
| Оранжевый | Личный кабинет администратора, управление |
| Розовый | Внешняя система (интеграция) |

---

## 1. Сайт-визитка

*Главная точка входа (фиолетовый узел)*

От неё ведутся разделы:

- **Контакты**
- **Тарифы и условия**
- **Количество свободных парковочных мест** (карта по секторам)
- **Страница регистрации нового клиента**
- **Страница авторизации** (клиент)
  - логин + пароль
  - SSO (Яндекс ID / VK ID / MAX)

---

## 2. Личный кабинет клиента (главная страница)

*Вход после авторизации клиента (зелёная ветка)*

**Содержимое главной страницы:**

- Статус парковки
- Текущий договор / тариф / сессия
- Статус оплаты / долг
- Количество свободных парковочных мест (карта по секторам)

**Разделы из ЛК клиента:**

- **Страница управления бронированием**
- **Страница управления договорами**
- **Страница «История парковочных сессий»**
- **Страница управления списком транспортных средств**
- **Страница настройки профиля**

---

## 3. Личный кабинет администратора

*Оранжевая ветка*

### 3.1 Вход

- **Страница авторизации личного кабинета админа** → вход в админ-раздел

### 3.2 Главная страница админа

- **Личный кабинет админа (главная страница)**
  - **Дашборд** с наиболее важными операционными данными / показателями

**Разделы из ЛК админа:**

- **Страница управления секторами и парковочными местами**
- **Страница управления тарифами**
- **Страница управления договорами**

### 3.3 Управление профилями и аналитика

От узла с операционными данными / показателями:

- **Страница управления профилями клиентов**
- **Страница управления профилями админов**
- **Страница с аналитикой (внешняя система)** — розовый узел, интеграция

### 3.4 Рабочий интерфейс охранника КПП

*По User Story Map и Impact Map — отдельный вход от ЛК админа.*

- **Страница авторизации рабочего интерфейса** (логин, пароль, код подтверждения) → интерфейс поддержки офлайн-клиентов, проверка доступа, операции на КПП.

---

## Схема связей (дерево)

```
Сайт-визитка
├── Контакты
├── Тарифы и условия
├── Количество свободных парковочных мест (карта по секторам)
├── Страница регистрации нового клиента
└── Страница авторизации (логин+пароль, SSO)
    └── Личный кабинет клиента (главная)
        ├── [Статус парковки, договор/тариф/сессия, оплата/долг, карта мест]
        ├── Страница управления бронированием
        ├── Страница управления договорами
        ├── Страница «История парковочных сессий»
        ├── Страница управления списком ТС
        └── Страница настройки профиля

Страница авторизации ЛК админа
└── Личный кабинет админа (главная)
    ├── Дашборд (операционные данные/показатели)
    ├── Страница управления секторами и парковочными местами
    ├── Страница управления тарифами
    ├── Страница управления договорами
    ├── Страница управления профилями клиентов
    ├── Страница управления профилями админов
    └── Страница с аналитикой (внешняя система)

Рабочий интерфейс охранника КПП (отдельный вход)
└── Авторизация (логин, пароль, код) → интерфейс КПП (доступ, исключения, офлайн-клиенты)
```

## Соответствие артефактам и протоколам

Проверка по User Story Map, Impact Map, Opportunity Canvas и протоколам №1.1–№5.

### Что совпадает

- **Сайт-визитка** (публичный портал без авторизации) — Контакты, Тарифы и условия, доступность мест; регистрация и авторизация клиента (логин+пароль, SSO). Соответствует протоколу №3 (сайт-визитка для неавторизованных), Impact Map (публичный портал), протоколу №4 (Business Site).
- **Личный кабинет клиента:** управление бронированием, договорами, история парковочных сессий, список ТС, настройки профиля; на главной — статус парковки, договор/тариф/сессия, оплата/долг, карта мест. Покрывает блоки 6–10 User Story Map и требования протокола №4 (Consumer App: бронирование, оплата, профиль).
- **Личный кабинет админа:** дашборд; управление секторами и парковочными местами (блок 11 USM), тарифами, договорами; управление профилями клиентов и админов (роли/доступы); аналитика как внешняя система. Соответствует Impact Map (админ-панель, CRM-функции, тарифы, шаблоны договоров, администрирование пользователей), протоколу №4 (админ-панель для владельца и управляющего). Протокол №4: «Аналитика в специальном виде не нужна на релизе» — в карте аналитика вынесена во внешнюю систему (интеграция/будущее).
- **Терминология:** тарифы, бронирование, договоры, ТС — в духе глоссария протокола №4 и User Story Map.

### Уточнения и расхождения

1. **Интерфейс охранника (КПП)** в п. 3.4 и в дереве добавлен как отдельный поток входа (по User Story Map и Impact Map — свой рабочий вход и интерфейс поддержки офлайн-клиентов).
2. **Владелец и управляющий** в карте объединены в «Личный кабинет админа». По артефактам: владелец — учётные записи, роли, права; управляющий — операционное управление (тарифы, договоры, места, клиенты). Это согласуется с протоколом №4 (одна админ-панель для обеих ролей с разграничением прав).
3. **Модуль обращений / обратная связь** (Impact Map, протокол №5 — «в ЛК возможность создать обращение, может быть не MVP») в карте явно не выделен. Может входить в ЛК клиента (например, в настройки профиля или отдельный раздел) или быть добавлен при детализации.
