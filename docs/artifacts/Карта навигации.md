# Карта навигации

Навигационная карта (sitemap) структуры сайта и личных кабинетов. Составлена по реальному UI (каталог `ui/`, шаблоны навигации и страницы) с учётом артефактов проекта; расхождения отражены в разделе «Отчёт по составлению карты навигации».

---

## Условные обозначения

| Цвет узла | Назначение |
|-----------|------------|
| Фиолетовый | Точка входа / главная (сайт‑визитка) |
| Зелёный | Публичные разделы и личный кабинет клиента |
| Оранжевый | Личный кабинет администратора и рабочие интерфейсы персонала |
| Розовый | Внешняя система (интеграция / аналитика) |

---

## 1. Сайт‑визитка (точка входа)

*Фиолетовый узел — публичный сайт и стартовая точка навигации.*

**Точка входа (index.html):**

- **Сайт парковки** → `site/index.html`
- **Личный кабинет клиента** → `site/login.html`
- **Вход для сотрудников** → `site/staff-login.html`

**Навигация сайта (nav-site):**

- **Главная** — `site/index.html` (описание парковки, компактный блок карты по секторам на главной)
- **Тарифы и условия** — `site/tariffs.html`
- **Правила парковки** — `site/rules.html`
- **Контакты** — `site/contacts.html`

**В шапке сайта:**

- **Регистрация** — `site/register.html`
- **Вход** — `site/login.html`

**В футере сайта:**

- **Вход для сотрудников** — `site/staff-login.html`

**Дополнительно:**

- Отдельная страница `site/parking-map.html` есть в UI, но в основном меню сайта ссылки на неё нет; карта по секторам выведена на главной странице (компактный блок).

## 2. Личный кабинет клиента

*Зелёная ветка — клиентская зона после успешной авторизации (вход через `site/login.html`).*

**Навигация ЛК клиента (nav-lk):**

- **Парковка** — `client/dashboard.html` (главная: быстрые действия, карта, тарифы; из блоков доступны: История парковочных сессий → `sessions-history.html`, Бронирования → `bookings.html`, Договоры → `contracts.html`, создание договора → `contracts-create.html`)
- **Оплаты** — `client/payment-settings.html`
- **Профиль** — `client/profile.html`
- Страницы без пункта в меню (переход с Парковки): `client/bookings.html`, `client/contracts.html`, `client/contracts-create.html`, `client/sessions-history.html`
- **Оплаты** — `client/payment-settings.html`
- **Обращения** — `client/appeals.html`
- **Профиль** — `client/profile.html`

**В шапке ЛК клиента:**

- Иконка **Уведомления** → `client/notifications.html`
- Имя пользователя → `client/profile.html`

**Дополнительные страницы (переходы с дашборда/разделов):**

- `client/contracts-create.html` — создание договора
- `client/contract-sign.html` — подписание договора кодом из SMS (после формирования договора); далее переход к оплате
- `client/payment-external.html` — внешняя оплата (переход из активной сессии / договора / раздела «Бронирование» / после подписания договора)

**Особенности навигации и адаптации ЛК клиента:**

- навигация ЛК (`nav-lk`) сворачивается в бургер‑меню на мобильных устройствах; состояние меню управляется атрибутом `data-nav` на `body` и кнопкой‑бургером в шапке;
- в футере ЛК есть переключатель версии интерфейса «для компьютера / для телефона», выбор сохраняется в `localStorage` (`lkView`) и отражается в атрибуте `data-view` на `body`; это влияет на представление интерфейса, но не создаёт отдельных страниц или узлов навигации.

## 3. Личный кабинет администратора

*Оранжевая ветка — административный интерфейс (вход через `site/staff-login.html`, выбор роли администратора).*

**Вход:** только через `site/staff-login.html` с выбором роли администратора; после выбора роли — переход на главную админки (`admin/index.html`).

**Навигация ЛК администратора (nav-role при role=admin):**

- **Главная** — `admin/index.html`
- **Секторы и ПМ** — `admin/sectors.html`
- **Тарифы** — `admin/tariffs.html`
- **Договоры** — `admin/contracts.html`
- **Бронирования** — `admin/bookings.html`
- **Задолженности** — `admin/debts.html`
- **Профили клиентов** — `admin/clients.html`
- **Профили админов** — `admin/admins.html`
- **Управление уведомлениями** — `admin/notifications-manage.html`
- **Обращения** — `admin/requests.html`
- **Аналитика** — `admin/analytics.html` *(розовый узел — внешняя система/интеграция)*

**Отдельный блок навигации (интерфейс КПП для администратора):**

- **КПП — панель** — `admin/kpp-panel.html`
- **КПП — журнал** — `admin/kpp-log.html`

**В шапке ЛК администратора:**

- Иконка **Уведомления** → `admin/notifications.html`
- Имя/профиль администратора → `admin/profile.html`

**Реестр договоров (admin/contracts.html):** в таблице в колонке «Действия» три кнопки — **Просмотр**, **Изменить**, **Расторгнуть**; они открывают соответственно модалки просмотра договора (`#admin-contract-modal`), редактирования (`#admin-contract-edit-modal`), расторжения (`#admin-contract-terminate-modal`). Блок расторжения вынесен из модалки просмотра в отдельную модалку.

**Дополнительные страницы и внутренние представления:**

- `admin/contracts-create.html` — создание договора
- `admin/client-sessions.html` — сессии клиента (переход из профиля клиента)
- `admin/debts.html` — страница задолженностей: сводка по задолженностям (в т.ч. блок «Просроченные оплаты» с кнопкой «Показать» → якорь на реестр `#debts-register`), реестр с фильтрами; действия «Договор» и «Карточка клиента» открывают модальные окна без перехода на отдельные страницы (`#admin-contract-modal`, `#admin-client-fl-modal`, `#admin-client-yl-modal`). В карточке клиента (ФЛ) доступно редактирование ТС: добавить / изменить / удалить через модальные окна (`#admin-client-vehicle-add-modal`, `#admin-client-vehicle-edit-modal`, `#admin-client-vehicle-delete-modal`).

## 4. Рабочий интерфейс охранника КПП

*Оранжевая ветка — отдельный контур; вход через `site/staff-login.html` с выбором роли охранника.*

**Навигация (nav-role при role=operator):**

- **Панель КПП** — `guard/index.html`
- **Журнал въезда‑выезда** — `guard/log.html`

**В шапке интерфейса охранника:**

- Иконка **Уведомления** → `guard/notifications.html`
- Имя/профиль сотрудника → `guard/profile.html` (в основном меню пункт «Профиль» не выведен)

**Переходы с панели и журнала:**

- Карточка клиента/ТС → `guard/client-summary.html`
- С карточки клиента → `guard/plate-history.html` (история по номеру)

**Выход:** возврат к `site/staff-login.html` (в текущем UI оформляется через профиль или контекст выхода, в шаблоне `nav-guard.njk` есть пункт «Выход», но в layout guard используется `nav-role.njk` без пункта «Выход» в основном меню).

## 5. Схема связей (дерево)

Текстовое дерево навигации по текущему UI:

```
[Точка входа (фиолетовый)]  index.html

  ├─ Личный кабинет клиента → site/login.html
  └─ Вход для сотрудников → site/staff-login.html

[Сайт‑визитка (фиолетовый)]  site/
  ├─ Главная (index.html), на главной — компактный блок карты по секторам
  ├─ Тарифы и условия (tariffs.html)

  ├─ Контакты (contacts.html)
  ├─ [шапка] Регистрация (register.html), Вход (login.html)
  ├─ [футер] Вход для сотрудников (staff-login.html)
  └─ parking-map.html (есть в UI, в основном меню нет ссылки)

[Личный кабинет клиента (зелёный)]  client/

  ├─ Бронирование (bookings.html)

[Личный кабинет администратора (оранжевый)]  admin/
  ├─ Вход (staff-login → выбор роли admin)
  ├─ Главная (index.html)
  ├─ Секторы и ПМ (sectors.html)
  ├─ Тарифы (tariffs.html)
  ├─ Договоры (contracts.html)
  │   ├─ [модалка] #admin-contract-modal (Просмотр)
  │   ├─ [модалка] #admin-contract-edit-modal (Изменить)

  ├─ Профили клиентов (clients.html)
  │   └─ client-sessions.html
  ├─ Профили админов (admins.html)
  ├─ Управление уведомлениями (notifications-manage.html)
```

## Отчёт по составлению карты навигации

Карта приведена в соответствие с реальной структурой UI. Источники: **Карта навигации (артефакт)** (предыдущая версия), **User Story Map**, **Концептуальная модель** (docs/artifacts/), **UI**: шаблоны `nav-site.njk`, `nav-lk.njk`, `nav-role.njk`, `nav-guard.njk`, layout'ы `site.njk`, `lk.njk`, `admin.njk`, `guard.njk`, страницы в `ui/site/`, `ui/client/`, `ui/admin/`, `ui/guard/`, партиалы `admin-contract-modal.njk`, `admin-contract-edit-modal.njk`, `admin-contract-terminate-modal.njk`, `admin-client-modal.njk`, `admin-client-vehicle-modals.njk`.

**Актуализация (по состоянию UI):** в навигации ЛК администратора добавлен пункт «Задолженности» (`admin/debts.html`). Реестр договоров: в колонке «Действия» три кнопки (Просмотр, Изменить, Расторгнуть) с отдельными модалками; расторжение вынесено из модалки просмотра. Страница задолженностей: сводка (в т.ч. «Просроченные оплаты», кнопка «Показать» → якорь на реестр), реестр с фильтрами, модалки договора и карточки клиента; в карточке клиента (ФЛ) — редактирование ТС (добавить/изменить/удалить) через модальные окна.

### Расхождения UI и артефактов

| № | Артефакт (Карта навигации / User Story Map / Концептуальная модель) | Реализация в UI | Примечание |
|---|----------------------------------------------------------------------|-----------------|------------|
| 1 | **Количество свободных парковочных мест** — отдельный раздел сайта‑визитки с картой по секторам | Отдельной страницы в основном меню нет; карта по секторам выведена компактным блоком на главной; есть отдельный файл `parking-map.html` без ссылки в nav-site | В артефакте — отдельный пункт навигации; в UI — только на главной и отдельная страница без входа в меню |
| 2 | **Платежи и счета** (ЛК клиента) | В меню ЛК: **Оплаты** (`payment-settings.html`) | Другое название раздела |
| 3 | **Настройки профиля** (ЛК клиента) | В меню ЛК: **Профиль** (`profile.html`) | Другое название; в артефакте акцент на «настройки» |
| 4 | **Уведомления** — отдельный пункт в основном меню ЛК клиента | Уведомления доступны только по иконке в шапке → `notifications.html`, в nav-lk пункта «Уведомления» нет | В артефакте — равноправный раздел меню; в UI — только иконка в шапке |
| 5 | **Правила парковки** | В UI есть отдельный пункт навигации **Правила парковки** (`rules.html`) в nav-site | В предыдущей карте навигации явно не выделялись; в UI — отдельная страница |
| 6 | **КПП** — рабочий интерфейс охранника как отдельная точка входа | В UI: отдельный контур `guard/` (Панель КПП, Журнал) и параллельно в админке блок «КПП — панель», «КПП — журнал» (`admin/kpp-panel.html`, `admin/kpp-log.html`) | Два способа доступа к функциям КПП: отдельный интерфейс охранника и ссылки из ЛК администратора |
| 7 | **Профиль охранника** | Есть `guard/profile.html`, доступ по ссылке из шапки (имя сотрудника); в основном меню nav-role пункта «Профиль» нет | Соответствует UI; в артефакте явно не детализировалось |
| 8 | **Уведомления** в ЛК администратора и у охранника | В обоих контурах — только иконка в шапке → `admin/notifications.html`, `guard/notifications.html`; в админке отдельный пункт меню — **Управление уведомлениями** (`notifications-manage.html`) | Лента уведомлений — по иконке; управление рассылками — отдельный раздел только в админке |
| 9 | Точка входа: «Авторизация администратора» и «Вход в рабочий интерфейс КПП» как отдельные ссылки с сайта | В UI одна страница **Вход для сотрудников** (`site/staff-login.html`), далее выбор роли (администратор / охранник) | Один вход для сотрудников с последующим выбором контура |
| 10 | Мобильная версия ЛК клиента | В UI реализованы бургер‑меню и переключатель «для компьютера / для телефона» в страницах ЛК клиента (`client/*.html`); из раздела «Бронирование» переход на внешнюю оплату (`payment-external.html`) | В предыдущих артефактах не описаны адаптивные сценарии ЛК клиента; учтено в карте как поведенческая особенность внутри контура ЛК без новых точек входа |

Перечисленные расхождения учтены при приведении структуры карты к UI. При изменении интерфейса карту навигации рекомендуется обновлять и при необходимости дополнять данный отчёт.
