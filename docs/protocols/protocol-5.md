## Ход встречи

### Event Storming TO-BE (согласованное видение)

**Мотивация регистрации в ЛК:** удобные способы оплаты и автосписание; скидка за регистрацию и онлайн-оплату; управление списком ТС; история сессий; оформление договора для длительного пользования на выгодных тарифах.

**Размещение информации:** тарифы, условия, оферта — информационные щиты, табло, сайт.

**Оплата:**

- ФЛ без ЛК и ФЛ почасовая — **постоплата** на выезде по фактическому времени парковочной сессии.
- ФЛ (договор) — оплата договора онлайн.

**Бесконтактный въезд–выезд:**

- Автоматическое распознавание номеров (ГРЗ).
- Автоматическая идентификация по ГРЗ: ФЛ (без ЛК) — разовый гостевой почасовой; ФЛ (почасовая, с ЛК); ФЛ (договор); ЮЛ.
- Автоматическое решение о доступе: проверка права по типу клиента/договору/тарифу; команда на открытие шлагбаума; фиксация факта проезда (въезд/выезд).
- **Парковочная сессия:** на въезде — создание (ГРЗ, тип клиента, сектор/ПМ, время начала); на выезде — завершение (время окончания, расчёт суммы, статус оплаты).
- Учёт занятости: на въезде — уменьшение доступности/фиксация занятого ПМ; на выезде — увеличение доступности/освобождение ПМ.

**Информационный дисплей на въезде:** ГРЗ; право доступа; тариф; инструкции; назначенное ПМ (ФЛ почасовая — сектор; ФЛ договор и ЮЛ — ПМ или диапазон по договору).

**Информационный дисплей на выезде:** ГРЗ; право доступа; тариф; сумма к оплате; статус платежа; инструкции. Шлагбаум на выезде открывается после подтверждения оплаты (или автосписания для клиентов с ЛК).

**Способы оплаты:**

- Онлайн: с ЛК — через ЛК + автосписание; все — по QR и вводу ГРЗ.
- Офлайн: терминал у шлагбаума на выезде (опция); терминал у охранника на КПП (резерв).

**Обработка исключений:**

- ГРЗ не распознан/некорректно: повторное распознавание; ручной ввод охранником.
- Право доступа не подтверждено: перевод в разовую почасовую (если разрешено) или отказ с инструкцией на дисплее и к охраннику.
- ГРЗ в чёрном списке: отказ или ручная обработка охранником.
- Ошибка/неподтверждённая оплата: шлагбаум не открывается; альтернативный способ (QR/ЛК/терминал); эскалация к охраннику.
- Недоступны внешние сервисы/оборудование: ручная обработка охранником; последующая синхронизация данных.
- Спорные ситуации: статус «спорная», фиксация основания; обработка по регламенту; при отказе оплаты — задолженность/инцидент, материалы для взыскания; при угрозе — эскалация, фиксация; для претензий/суда — сохранение доказательной базы (сессия, оплата, журнал СКУД, камеры).

**Позиция заказчика:**

- Согласовано: постоплата; оплата через ЛК как основной онлайн-сценарий; договор закрепляет ПМ (или диапазон) на весь срок; понятный интерфейс на въезде (ГРЗ, тариф, право доступа, инструкции); явное согласие с офертой/правилами.
- Согласовано с оговорками: распознавание ГРЗ — предпочтительный механизм (учёт влияния на пробки, стоимость/альтернативы); терминал у шлагбаума на выезде — возможен если недорого, не обязателен (терминал у охранника есть).
- **Не согласовано:** почасовые ФЛ без регистрации/контактов — заказчик настаивает на **минимальной идентификации (телефон + имя)**; множить онлайн-каналы оплаты в MVP (ЛК + QR) — из-за усложнения сценариев.
- Открытый вопрос: в MVP только ЛК или ЛК + QR?
- Предложение заказчика: терминал/кнопка подтверждения согласия с тарифом и правилами на въезде для ФЛ без ЛК как «явный акт согласия».

### Opportunity Canvas (правая часть)

**Идеи решений:** ЛК клиента; админ-панель; публичный портал/сайт; интеграция с онлайн-оплатой; система обращений (вероятно не MVP); биллинг, 1С (не MVP); СКУД, LPR-камеры, терминал офлайн (резерв), информационные табло.

**Ценность для пользователей:** переформулированные воздействия из Impact Map. Заказчик уточнил: в ЛК должна быть возможность создать обращение (может быть не MVP).

**Пользовательские метрики:** значения скорректированы с заказчиком; заказчик может добавить/уточнить метрики асинхронно.

**Стратегия внедрения (согласовано):** перевод клиентов в онлайн и мотивация (в т.ч. рекламные активности); обучение персонала (СКУД, камеры, спорные ситуации); обучение управляющего/заказчика (админ-панель, показатели, настройки, права, оплаты).

**Бизнес-метрики:** Рейтинг на геосервисах — не ниже 4,5. Доля клиентов «через онлайн» — из цели Impact Map. Пропускная способность — расчёт с поправочным коэффициентом на новых клиентов и спорные/проблемные случаи; текущая 40–70 машин/час. Простой — снизить до <10%.

**Бюджет:** Команда: разработка (сайт/ЛК/админка) 4–6 мес — 5–10 млн руб. (без оборудования). Заказчик: ориентир **~10 млн руб. на всё (включая оборудование)**, при обосновании готов рассмотреть до **15 млн**.

### Договорённости

**От команды:** обновить Event Storming TO-BE с учётом минимальной идентификации (телефон + имя); проработать вариант «явного акта согласия» для ФЛ без ЛК (терминал/кнопка), описать сценарий; предложение по составу MVP по оплате (только ЛК vs ЛК + QR) на согласование; уточнить состав и стоимость оборудования (LPR, табло, терминалы) и влияние на бюджет; обновить правую часть Opportunity Canvas и отправить заказчику.

**От заказчика:** обратная связь по онлайн-каналу(ам) оплаты в MVP; подтвердить требования к минимальной идентификации (состав данных, точка сбора); обратная связь по варианту «явного согласия» на въезде; при необходимости — дополнить метрики в чате.

**Общее:** зафиксировать состав MVP по оплате и гостевому сценарию после обмена в чате; согласование артефактов асинхронно; обсудить DoD для User Story Map в части MVP в чате.
