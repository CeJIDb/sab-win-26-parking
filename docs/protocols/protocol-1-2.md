## Ход встречи

### Текущие бизнес-процессы (AS-IS)

#### Въезд на парковку

**Подпроцессы:** Идентификация клиента/ТС; Проверка права доступа на въезд.

**Идентификация:** по проксимити-карте; по данным клиента (вручную); по номеру ТС (для ЮЛ); через подтверждение управляющего/представителя компании по телефону.

**Проверка права доступа:** наличие и срок действия договора; статус оплаты/задолженности; корректность и работоспособность карты; принадлежность ТС (для ЮЛ); ограничения и «чёрный список».

**Сценарии:**
- **Новый клиент — ФЛ:** краткосрочная (до суток) — договор не заключается, данные в Excel, оплата охраннику, шлагбаум вручную; оформление абонемента — старший охранник, бумажный договор, оплата (до 2 мес — полная сумма, от 2 мес — по частям), выдача проксимити-карты, открытие по карте автоматически.
- **Новый клиент — ЮЛ:** условия согласовываются с управляющим, бумажный договор, подпись Владельца, данные и номера ТС в учётную таблицу, оплата по условиям договора (возможна отложенная/поэтапная), доступ после идентификации и проверки права.
- **Действующий клиент — ФЛ:** карта к считывателю → идентификация и проверка → шлагбаум автоматически или при ошибке — ручная проверка, перепрограммирование/выдача новой карты; при забытой карте — идентификация, проверка, шлагбаум вручную (временная карта не выдаётся); при утере — новая карта за счёт клиента.
- **Действующий клиент — ЮЛ:** камера только для визуального контроля, автоматическое распознавание номеров отсутствует; охранник проверяет номер по учётной таблице; номер в базе → шлагбаум вручную; номера нет → идентификация, связь с управляющим/представителем, внесение ТС в таблицу, шлагбаум вручную. При невозможности подтверждения — проезд не разрешается.

#### Выезд с парковки

**Идентификация:** по проксимити-карте; по данным клиента (вручную); по номеру ТС (визуально).

**Проверка права на выезд:** факт оплаты, задолженность, корректность карты, принадлежность ТС (ЮЛ), ограничения/чёрный список.

**Сценарии:** автоматический выезд по карте; ручной при отсутствии/неисправности карты. При краткосрочной парковке — проверка оплаты, при задолженности — доплата; при отказе от доплаты — внесение в «чёрный список» (ведётся вручную по клиенту, не по номеру авто).

#### Навигация по парковке

Парковка разделена на сектора, есть навигационные указатели и нумерация мест. Проблемы: занятое место другим авто, место недоступно из-за ремонтов — клиент обращается к охраннику. Часть клиентов самостоятельно занимают свободные места без согласования.

#### Оплата

**ФЛ:** охранник принимает наличными/терминалом, вручную рассчитывает стоимость, выдаёт счёт и чек, фиксирует в Excel.

**ЮЛ:** по счёту; бухгалтер получает информацию об оплатах из банка.

#### Заключение договоров

**ФЛ:** договор заполняется старшим охранником по шаблону, распечатывается, клиент проверяет и подписывает, затем подписывает управляющий; экземпляр клиенту при следующем визите. Пролонгации нет — новый договор. Паспортные данные в базе не хранятся.

**ЮЛ:** процесс не детализирован (частично см. въезд — новый клиент ЮЛ).

#### Ведение и актуализация учётных данных

**ФЛ:** охранник фиксирует оплаты и договоры вручную; при задолженности/окончании договора карта блокируется старшим охранником.

**ЮЛ:** бухгалтер передаёт данные об оплатах управляющему, управляющий вручную вносит в учётные таблицы.

#### Обслуживание инфраструктуры

Охранник по заданию управляющего проводит осмотр (шлагбаумы, освещение, камеры, покрытие, разметка), составляет отчёт; управляющий решает — своими силами или подрядчик; работы подрядчика принимаются по акту.

### Зафиксированные особенности текущего состояния

- Нет единого формализованного источника данных; проверка права доступа вручную/полуавтоматически; решения охранников на месте без формализованных регламентов; сильная зависимость от человеческого фактора.
- Учётные таблицы — единственный источник; актуальность зависит от ручного внесения; по ЮЛ — задержки из-за передачи данных бухгалтер ↔ управляющий.
- Контроль доступа не автоматизирован, не интегрирован с договорами и оплатами; блокировка карт вручную; нет централизованного контроля сроков и оповещений; чёрный список вручную, не привязан к номеру ТС.
- Видеонаблюдение только для визуального контроля; автоматическое распознавание номеров отсутствует; шлагбаумы не интегрированы с учётными таблицами.
- Клиенты не получают оперативную информацию о свободных местах; при спорах — обращение к охране; нестандартные ситуации решаются индивидуально.

### Основной бизнес-процесс (из комментариев)

**Предоставление парковочного места** — основной (end-to-end) процесс от въезда до выезда.

**Подпроцессы:** 1) Идентификация клиента; 2) Оформление клиента (заключение договора); 3) Оплата услуг парковки; 4) Навигация и размещение на парковке; 5) Выезд с парковки.

**Поддерживающий процесс:** Актуализация прав доступа на парковку — актуальность данных о договорах, оплатах и доступах (выполняется вне конкретного клиентского сценария).
