## Ход встречи

### Контекст и текущая состояние парковки

- **Расположение:** Санкт-Петербург, окружение: жилые дома, метро, два учреждения (НИИ и статистическая служба), супермаркет со своей бесплатной парковкой.
- **Характеристики парковки:**
  - Открытого типа, одноэтажная, круглосуточная (24/7)
  - Вместимость — 600 мест, из которых 50 мест для людей с ОВЗ
  - Зонирование: условное разделение на зоны для краткосрочной и долгосрочной аренды (визуально не отличается), места для людей с ОВЗ (место больше по размеру и ближе к выходу), выделенная зона для подрядчика с техникой
  - Возможность парковки мотоциклов (до 3 на место)
- **Инфраструктура:**
  - 2 КПП, старые шлагбаумы, проксимити-карты
  - Видеонаблюдение (замкнутый контур)
  - Интернет (проводной)
  - Возможность монтажа оборудования

### Организационная структура и персонал

- **Сотрудники:**
  - Владелец бизнеса (Елена Гриценко)
  - Охрана: 2 охранника + 1 старший на каждом КПП, смены по 12 часов (всего в штате охраны — 12 человек)
  - Управляющий (операционный директор, работает днем, ведёт корпоративных клиентов)
  - Бухгалтер (на аутсорсе, работает в 1С)
- IT-специалистов или персонала по техническому обслуживанию в штате нет (все текущие интегрированные инфосистемы были поставлены сторонними вендорами).
- **Ответственность:** владелец — стратегия и управленческие решения; управляющий — текущие/ситуационные вопросы на парковке.

### Бизнес-процессы как есть (AS-IS)

**Учёт данных:**
- Ведётся в Excel: база клиентов, база договоров, журнал въезда/выезда
- Бэкап раз в месяц на внешний носитель
- Данные не интегрированы, аналитика затруднена
- Основная работа — на охранниках и управляющем

**Въезд:**
- Осуществляется через старый шлагбаум
- Шлагбаум открывается по проксимити-карте (автоматически) или вручную (при проблемах с картами или краткосрочной парковке до суток)
- Если карта не сработала: идентификация клиента охраником, проверка договора и оплаты; выдача/перепрограммирование/замена карты
- Клиент занимает закреплённое за ним по договору место парковки (назначение конкретного места только для долгосрочной; при краткосрочной место указывает охранник лично)

**Оплата:**
- Оплата производится охраннику (наличные / терминал)
- Предоплата (с доплатой при перелимите)
- Отсутствие возврата при досрочном выезде

**Выезд:**
- По карте или вручную
- Проверка оплаты (при необходимости)

**Договоры:**
- Бумажные (заполняются в Word, подписываются вручную)
- Заключает старший охранник (критические 20 минут)
- Подписывает управляющий (по доверенности)
- Для юр. лиц подписывает владелец

### Основные проблемы

- Очереди на въезде/выезде (из-за ручного управления, старого оборудования)
- Перегрузка охранников (не их профильная работа: консультирование, оплата, договоры)
- Отсутствие аналитики у собственника (данные в Excel, трудно обрабатывать)
- Неудобства для клиентов (необходимость приезжать на парковку для получения информации)
- Около 30% мест не используется → потери дохода
- Расходы на замену проксимити-карт
- Риск потери клиентов из-за плохого пользовательского опыта

### Цели и ожидания от проекта

**Цели:**
- Быстрый въезд на парковку (до 1 минуты)
- Сокращение рутинных операций охранников с 80% до 20%
- Получение аналитики данных для управленческих решений

**Желаемый результат:**
- Автоматизация въезда/выезда (возможная замена шлагбаума и карт)
- Онлайн-платформа для клиентов (бронирование, оплата, просмотр тарифов)
- Личный кабинет / удалённый доступ
- Аналитика для владельца (по клиентам, тарифам, загрузке, финансам)
- Возможность дополнительных услуг за доп. плату (наблюдение за ТС онлайн)
- Снижение процента неиспользуемых мест → увеличение дохода

**Ключевые метрики успеха:**
- Увеличение доли взаимодействия клиентов с парковкой через онлайн-системы
- Сокращение количества неиспользуемых мест
- Снижение рутинных задач охранников с 80% до 20%
- Ускорение въезда (процесс не более 1 минуты)

**Критерии неудачи:**
- Сохранение очередей
- Отсутствие аналитики
- Сохранение ручных операций охранников

### Ограничения и требования

**Технические:**
- Обязательно законное и лицензированное ПО, зарегистрированное в реестре российского ПО
- Безопасность данных, соответствие 152-ФЗ
- Возможность авторизации через Яндекс ID, ВК ID, Госуслуги

**Процессные:**
- Не трогать работу бухгалтера
- Подписание договоров: управляющий (по доверенности) для физлиц, владелец — для юрлиц

**Сроки:** первый результат — за 4 месяца, максимум — 6. Масштабируемость на другие парковки.

**Бюджет:** нет фиксированной суммы, обсуждается при обосновании затрат.

**Внедрение:** предпочтение — итеративное (поставки частями).
