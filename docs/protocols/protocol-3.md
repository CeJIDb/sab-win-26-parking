- **Выезд:** идентификацию оформить как подпроцесс; не углубляться в мелкие операционные задачи.

### Opportunity Canvas

### Impact Map

**Цель проекта:** увеличить долю клиентов, использующих онлайн-сервисы, до **80%** от общего количества клиентов на дату запуска. Отсчёт с момента запуска, срок достижения — **6 месяцев**.

**Акторы:** юрлица, физлица, охранники, управляющий, владелец. Масштабирование — один управляющий управляет несколькими парковками. Подрядчики — минимальное участие.

**Воздействия (уточнения):**
- Управление ТС: подавать и редактировать список ТС — и для ФЛ, и для ЮЛ; объединить карточки.
- Информирование клиентов — обобщённо, без детализации каналов.
- Выставление счетов — не в первом приоритете.
- Инциденты → «создание обращения / поддержка / обратная связь».
- Бронирование — онлайн для ФЛ.
- Ключевой принцип — въезд и выезд без взаимодействия с персоналом.
- Аналитика: объединить «метрики для маркетинга» и «данные для аналитики»; «Сверка поступивших оплат» → управление доступом; Аудит — история использования, объединить с историей для ФЛ.
- Онлайн-оплата — обязательная возможность.
- Обратная связь — объединить с обращениями.
- Управление задолженностями: автоматические напоминания, возможность оплаты, графики платежей, уведомления.
- Договоры в онлайн: заключить → отслеживать статус → продлить.

**Поставка / Решения:**
- **Для авторизованных клиентов:** Личный кабинет; онлайн-бронирование/выбор места; самообслуживание по спискам ТС; конструктор договоров (требуется).
- **Для неавторизованных:** Сайт-визитка; геосервисы.
- **Для управляющего и персонала:** CRM; Панель администратора (управление шаблонами договоров, тарифами, сотрудниками); Модуль обращений/поддержки; Отчётность/аналитика.
- **Интеграции:** Биллинг/платежи (полноценный документооборот — только для ЮЛ; для ФЛ — упрощённая модель); ЭДО (риск разных провайдеров, нужно универсальное решение); Контроль доступа (СКУД, бесконтактный въезд/выезд + запасной сценарий при отказе).

**Приоритеты:**
- **Высший приоритет:** Всё, что устраняет очереди и ручную работу охранников. Ключевая функция — бесконтактный въезд/выезд.
- **Обязательный функционал:** Личный кабинет (заключение, отслеживание, оплата), панель администратора (CRM, тарифы, шаблоны, сотрудники), QR-код на КПП.
- **Требует проработки:** Биллинг, интеграция с ЭДО, размещение на геосервисах, сайт-визитка.
