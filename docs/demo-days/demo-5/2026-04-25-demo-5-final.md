# План: подготовка к финальному Демо 5

**Дата:** 2026-04-25
**Задача:** подготовить выступление команды Парковка на финальном Демо 5 — 45 минут доклада + 15 минут Q&A — со сборкой материалов, закрытием долгов по ДЗ и прогонами.
**Время:** ~30 часов командной работы в течение недели до даты демо, распределенных по 5 спикерам.

Регламент демо — [demo-days-overview.md](../demo-days-overview.md#демо-день-5-финальное-демо).

## Связанные документы

- [talk-structure.md](./talk-structure.md) — структура доклада, тайминг, что резать, фразы-мостики между спикерами.
- [qa-prep.md](./qa-prep.md) — распределение зон Q&A, правило «5 секунд», 23 вопроса FAQ.
- [stage-setup.md](./stage-setup.md) — техника на сцене, чек-лист дня демо.
- [no-brokers-pitch.md](./no-brokers-pitch.md) — материал к ADR-слайду «асинхронный обмен без брокеров».
- [demo-5-script.md](./demo-5-script.md) — тексты выступлений по 5 спикерам (создается в фазе 0.2).

## Цель за неделю

К дате демо у команды должно быть:

- доклад на 38 минут речи + 7 минут буфер = 45 минут на сцене, с ~30 слайдами в Google Slides + навигационная доска в Miro;
- готовые тексты выступлений по 5 спикерам в [demo-5-script.md](./demo-5-script.md);
- закрытые критичные долги ДЗ, без которых блоки доклада «дырявые» (см. фазу 1);
- 2 полных прогона с фиксацией времени и правками;
- backup-материалы в Miro для Q&A: SQL, DRAKON, полные контракты, детальные требования к брокерам.

## Scope

### Входит

- Сборка каркаса в [./](./): `readme.md`, `demo-5-script.md`, `slides/`.
- Закрытие критичных долгов из [parking-hw-report-cejidb.md](../../process/parking-hw-report-cejidb.md) — приоритеты в фазе 1.
- Тексты выступлений по 5 спикерам, по блокам.
- Слайды в Google Slides (~30 шт).
- Навигационная доска Miro со ссылками на каноничные артефакты и backup-материалы.
- 2 прогона + правки.
- Ретроспектива по итогам в `docs/process/retro/YYYY-MM-DD-demo-5-final.md` (правило 6 из CLAUDE.md).

### Не входит

- Изменения в [docs/specs/](../../specs/) (требований не трогаем без отдельного согласования — правило 3 из CLAUDE.md).
- Новые wireframe в [ui/](../../../ui/) (используем существующее).
- Замена готовых артефактов из этапов 1–4: их только дорабатываем в случае пометок «Сдано, доделать».
- Артефакты по брокерам сообщений (Kafka, Rabbit) не делаем: в нашей архитектуре брокеров нет — выбран модульный монолит с Outbox-паттерном для уведомлений и локальным буфером для LPR/СКУД. См. [adr-003-modular-monolith.md](../../architecture/adr/adr-003-modular-monolith.md). На демо показываем именно это решение.

## Правила коммитов и веток

- Ветка: `docs/demo-5-final`.
- Коммиты в стиле Conventional Commits, scope `demo-5`. Примеры:
  - `docs(demo-5): добавить каркас demo-5 и индексный readme`
  - `docs(demo-5): добавить тексты выступления skifup по блокам 0, 1, 6`
  - `docs(demo-5): сверстать слайды этапа 5а`
- Атомарные коммиты при крупном диффе через `npm run commit:atomic`.
- Долги ДЗ — отдельные коммиты в той же ветке или в специализированных ветках по правилам [CONTRIBUTING.md](../../../CONTRIBUTING.md), на усмотрение владельцев артефактов.
- Каждый член команды коммитит **только свои** изменения (правило 1 из CLAUDE.md).
