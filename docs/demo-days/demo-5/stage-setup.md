# Финальное Демо 5 — техника на сцене

Чек-лист открывается за полчаса до выхода на сцену. План подготовки — [2026-04-25-demo-5-final.md](./2026-04-25-demo-5-final.md).

Демо — это не «показал слайд, рассказал». Это связка: проектор + ноутбук cejidb + Google Slides + Miro + интернет.

## Принципы

- **Шарит экран только cejidb.** Один источник, один Wi-Fi, одна проверенная техника. Остальные говорят со своих мест по сценарию, без переключения ноутбука.
- **Локальные копии всего, что показываем.** PDF-экспорт Google Slides на ноутбуке cejidb (не только в облаке). Каноничные артефакты из `docs/` открыты в IDE заранее. Скриншоты ключевых Miro-блоков — в [./slides/](./slides/) или внутри той же PDF.
- **Miro — bonus, не критическая зависимость.** Если Miro не отдает — продолжаем по PDF и `docs/`. У cejidb должна быть mental model «как вести демо без Miro» на случай отвала.
- **Проверка на той же технике.** Ноутбук cejidb + проектор/второй экран + адаптер — проверить в реальной аудитории или с тем же оборудованием, не дома. Минимум один проход «слайды → Miro → docs/» перед прогоном 3.
- **Резерв на интернет.** Запасной канал — телефон cejidb с раздачей. Заранее проверить, что ноутбук подключается без ввода пароля «на сцене». Зарядка телефона и ноутбука — обязательна.

## Чек-лист в день демо (cejidb)

- [ ] Кабель/адаптер к проектору, запасной кабель если есть.

- [ ] Открытые вкладки: Google Slides, Miro, `docs/` в IDE.
- [ ] Доступ к раздаче с телефона проверен на ноутбуке без ввода пароля «на сцене».
- [ ] Минимум один проход «слайды → Miro → docs/» на той же технике сделан.
