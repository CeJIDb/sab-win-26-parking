## Слайд 1. Контекстная диаграмма

Первый слайд работает как титульный вход в блок.
Он должен быть максимально простым:

- крупный заголовок по центру;
- без дополнительных карточек и перегрузки;
- задача слайда: ввести тему блока.

```mermaid
flowchart TB
    T["Контекстная диаграмма"]

    classDef title fill:#ffffff,stroke:#ffffff,color:#a86f43,font-size:34px,font-weight:bold;
    class T title;
```

## Как это лучше перенести на реальный слайд

- Оставить только заголовок `Контекстная диаграмма`.
- Не добавлять подпояснений и списков.
- Использовать слайд как короткую отбивку перед содержательной частью.

## Слайд 2. Четыре ключевые пользы

Ниже черновой макет второго слайда в формате `Mermaid`.
Он показывает композицию слайда с четырьмя карточками пользы:

- заголовок сверху;
- 4 блока в сетке `2x2`;
- короткие формулировки внутри карточек.

```mermaid
flowchart TB
    T["Четыре ключевые пользы"]

    subgraph R1[" "]
        direction LR
        A["01<br/><b>Границы ответственности</b><br/><br/>Показывает, что входит в платформу, а что остаётся вне ее зоны ответственности."]
        B["02<br/><b>Внешнее окружение</b><br/><br/>Фиксирует, кто и какие внешние системы взаимодействуют с платформой."]
    end

    subgraph R2[" "]
        direction LR
        C["03<br/><b>Ключевые потоки данных</b><br/><br/>Делает видимыми основные входящие и исходящие потоки данных через границы системы."]
        D["04<br/><b>Основа для проектирования</b><br/><br/>Служит базой для Use Case, концептуальной модели данных и функциональных требований."]
    end

    T --> R1
    R1 --> R2

    classDef title fill:#ffffff,stroke:#ffffff,color:#a86f43,font-size:28px,font-weight:bold;
    classDef card fill:#eef0ff,stroke:#c89b3c,color:#111111,stroke-width:1px,font-size:16px;
    classDef group fill:#ffffff,stroke:#ffffff,color:#ffffff;

    class T title;
    class A,B,C,D card;
    class R1,R2 group;
```

## Практические советы по переносу в слайд

- Делайте сетку `2x2`, а не 4 карточки в одну линию.
- Заголовок лучше оставить коротким и крупным: `Четыре ключевые пользы`.
- В каждой карточке оставьте:
  - большой номер;
  - короткий заголовок;
  - 2-3 строки описания.

## Слайд 3. Границы системы и внешнее окружение

Ниже макет третьего слайда в формате `Mermaid`.
Он показывает композицию, соответствующую текущему `jpg`-слайду:

- слева два блока внешнего окружения;
- справа платформа как единая система;
- между ними связи взаимодействия.

```mermaid
flowchart LR
    subgraph L1["Акторы"]
        direction TB
        CL["Клиенты: ФЛ и ЮЛ"]
        ST["Сотрудники: охранники,<br/>управляющий, владелец"]
    end

    subgraph L2["Внешние системы и оборудование"]
        direction TB
        EQ["Оборудование на объекте:<br/>СКУД, терминал, табло и дисплеи"]
        DIG["Внешние цифровые сервисы:<br/>платежная система, ОФД, SSO,<br/>уведомления, геосервисы и аналитика"]
        CORP["Корпоративные системы:<br/>ЭДО и 1С"]
    end

    subgraph R["Цифровая платформа парковки"]
        direction TB
        BS["Сайт-визитка"]
        CA["Личный кабинет"]
        AP["Панель администратора"]
        GI["Интерфейс охранника на КПП"]
        BL["Бизнес-логика"]
    end

    L1 --> R
    L2 --> R

    classDef actor fill:#eef0ff,stroke:#c89b3c,color:#111111,stroke-width:1px,font-size:16px;
    classDef system fill:#f8f8f8,stroke:#a86f43,color:#111111,stroke-width:2px,font-size:16px;
    classDef ext fill:#ffffff,stroke:#8aa0b8,color:#111111,stroke-width:1px,font-size:15px;
    classDef group fill:#ffffff,stroke:#ffffff,color:#444444,font-size:18px,font-weight:bold;

    class CL,ST actor;
    class BS,CA,AP,GI,BL system;
    class EQ,DIG,CORP ext;
    class L1,L2,R group;
```

## Как это лучше перенести на реальный слайд

- Слева разместить 2 карточки:
  - `Акторы`
  - `Внешние системы и оборудование`
- Справа разместить контейнер `Цифровая платформа парковки`.
- Между ними провести 2 стрелки взаимодействия.

## Слайд 4. Ключевые потоки данных

Ниже макет четвертого слайда в формате `Mermaid`.
Он показывает рекомендуемую композицию для слайда про основные потоки данных через границы платформы:

- внешние сущности слева;
- платформа в центре;
- укрупнённые потоки справа.

```mermaid
flowchart LR
    subgraph L["Внешние сущности"]
        direction TB
        CLIENT["Клиенты"]
        STAFF["Сотрудники"]
        ACS["СКУД + LPR"]
        PAY["Платёжная система + ОФД"]
        NOTIF["Сервис уведомлений"]
        DOC["ЭДО / 1С"]
    end

    CORE["Цифровая платформа парковки"]

    subgraph R["Ключевые потоки данных"]
        direction TB
        F1["Клиентские и операционные данные"]
        F2["Контур доступа через СКУД"]
        F3["Финансовый и сервисный контур"]
    end

    CLIENT --> CORE
    STAFF --> CORE
    ACS --> CORE
    PAY --> CORE
    NOTIF --> CORE
    DOC --> CORE

    CORE --> F1
    CORE --> F2
    CORE --> F3

    classDef ext fill:#eef0ff,stroke:#c89b3c,color:#111111,stroke-width:1px,font-size:15px;
    classDef system fill:#f8f8f8,stroke:#a86f43,color:#111111,stroke-width:2px,font-size:18px,font-weight:bold;
    classDef flow fill:#ffffff,stroke:#8aa0b8,color:#111111,stroke-width:1px,font-size:15px;
    classDef group fill:#ffffff,stroke:#ffffff,color:#444444,font-size:18px,font-weight:bold;

    class CLIENT,STAFF,ACS,PAY,NOTIF,DOC ext;
    class CORE system;
    class F1,F2,F3 flow;
    class L,R group;
```

## Рекомендуемая таблица для слайда

| Внешняя сущность | В платформу | Из платформы |
|---|---|---|
| Клиенты | Профиль, ТС, параметры бронирования и договора | Тарифы и правила, статусы доступа, сессии и оплаты, уведомления |
| Сотрудники | Параметры тарифов, секторов, договоров и бронирований | Статусы доступа, данные по сессиям и оплате |
| СКУД + LPR | ГРЗ, дата-время, КПП, въезд/выезд | Решение о доступе: разрешен/запрещен, команда на открытие |
| Платёжная система + ОФД | Статус платежа, статус фискализации, фискальный чек | Сумма к оплате, запрос на оплату, чек к фискализации |
| Сервис уведомлений | Статус отправки | Уведомление, канал, адресат |
| ЭДО / 1С | Статусы документов/сверки | Выгрузки по оплатам и сессиям |

## Слайд 5. От контекстной диаграммы к Use Case

Ниже макет пятого слайда в формате `Mermaid`.
Он нужен как короткий переход от блока про контекстную диаграмму к следующему разделу презентации:

- что было зафиксировано на уровне контекста;
- что это дало для анализа;
- почему дальше логично перейти к `Use Case`.

```mermaid
flowchart TB
    T["От контекстной диаграммы к Use Case"]

    subgraph R1[" "]
        direction LR
        A["Что зафиксировали<br/><br/>• границы платформы<br/>• внешнее окружение<br/>• потоки данных"]
        B["Что это дало<br/><br/>• ключевые сущности и сценарии взаимодействия<br/>• внешние интеграции<br/>• зону ответственности платформы"]
    end

    T --> R1

    classDef title fill:#ffffff,stroke:#ffffff,color:#a86f43,font-size:28px,font-weight:bold;
    classDef card fill:#eef0ff,stroke:#c89b3c,color:#111111,stroke-width:1px,font-size:16px;
    classDef group fill:#ffffff,stroke:#ffffff,color:#ffffff;

    class T title;
    class A,B card;
    class R1 group;
```

## Как это лучше перенести на реальный слайд

- По центру сделать 2 смысловых блока:
  - `Что зафиксировали`
  - `Что это дало`
- Между ними добавить стрелку или визуальную связь слева направо.
