# ERD (Entity-Relationship Diagram)

В этой папке собраны материалы по ER-модели проекта: общая нормализованная модель и вспомогательные импортированные материалы.

## Оглавление

- [Ключевые документы](#ключевые-документы)
- [Импортированные материалы](#импортированные-материалы)
- [Контексты ревью](#контексты-ревью)
- [Заметки](#заметки)

## Ключевые документы

- [Нормализованная ER-модель](erd-normalized-er-model.md) — полный нормализованный ERD в человекочитаемом виде.
- [DDL ERD для PostgreSQL (`public`)](../../../../sql/database/chartdb-postgresql-erd-normalized-public.sql) — актуальный SQL-экспорт схемы, по которому сверяется ERD-документация.

## Импортированные материалы

Импортированные материалы используются как вспомогательный контекст при сверке модели данных.
Они не являются каноничным источником истины для ER-модели.

- [Импортированный словарь данных из BuildIn](data-dictionary-buildin-import.md) — экспортированный словарь данных, приведенный к формату репозитория.

## Контексты ревью

Контексты ревью — вспомогательные материалы из обсуждений. Они **не являются источником истины** для ER-модели: каноничные состояния фиксируются в `erd-normalized-er-model.md` и сверяются с `chartdb-postgresql-erd-normalized-public.sql`.

На текущий момент отдельные файлы рабочих review-контекстов в репозитории не хранятся. Если они понадобятся повторно, их нужно заново оформить как артефакты рядом с ERD-документацией и добавить в это оглавление.

## Заметки

- **Аудит:** в целевой БД у **каждой** таблицы предполагаются `created_at` и `updated_at` (`TIMESTAMPTZ NOT NULL DEFAULT now()`; `updated_at` обновляется триггером `moddatetime`). В каноничном артефакте `erd-normalized-er-model.md` эти поля перечислены явно для наглядности и DrawSQL; в обзорной Mermaid внутри документа они опущены.
- При актуализации ERD сначала сверяйте документ с [DDL ERD для PostgreSQL (`public`)](../../../../sql/database/chartdb-postgresql-erd-normalized-public.sql), затем обновляйте человекочитаемое описание.
- Ссылки внутри папки — относительные (чтобы переносы были дешевле).
- Кросс-контекстные связи (между схемами) документируются как **логические ссылки** (без `REFERENCES`), см. ADR-003.
