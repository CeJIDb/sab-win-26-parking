# Стенограмма встречи 04.02.2026

**Стенограмма встречи: BPMN, Opportunity Canvas, Impact Map**

**Участники:** Александр Друзякин (ведущий), Артём Субботин (секретарь), Павел Борунов (хронометрист), Марина Бугаевская, Денис Салмин; Елена Гриценко (заказчик).

**Тема:** Замечания по BPMN и Opportunity Canvas; обсуждение Impact Map (цель, акторы, приоритеты, поставка).

---

## Ключевые темы (по ходу встречи)

- BPMN: оплата (управляющий, два экземпляра чека); заключение договора — два процесса (ФЛ и ЮЛ); выезд — идентификация как подпроцесс
- Opportunity Canvas: формулировки без жаргона, «недополученный доход», выделение жирным
- Impact Map: цель 80% онлайн за 6 мес., стоп-кадр на дату запуска, 20% — запасная ветка
- Акторы, влияния, поставка (Deliverables)
- Зоны ответственности: владелец — только договоры с ЮЛ; остальное — управляющий

---

## Фрагменты стенограммы (ключевые реплики)

**BPMN:** По оплате — обновления (управляющий получает данные для бухгалтерии; факт оплаты/неоплаты, список задолженностей); два экземпляра чека (клиент + архив). Заключение договора — ФЛ и ЮЛ разнести в два отдельных процесса для читаемости. Выезд: идентификацию оформить как подпроцесс; при наличии карты — приложил, уехал; ручная идентификация только для почасовых и корп-клиентов.

**Opportunity Canvas:** «Узкое горло» переформулировать (без жаргона). «Финансовые потери из-за простоя» → «недополученный доход». Выделение ключевых проблем жирным — хорошая практика. Материалы AS-IS согласованы с учётом комментариев.

**Impact Map — цель:** Увеличить долю клиентов, использующих онлайн-сервисы, до 80% от количества клиентов на дату запуска; срок достижения — 6 месяцев. Стоп-кадр на дату запуска (учёт сезонности, «чистый лист»). Остальные 20% — те, кому тяжело с цифрой; запасная ветка, охранники для нестандартных ситуаций.

**Impact Map — акторы:** Юрлица, физлица, охранники, управляющий, владелец; масштабирование — один управляющий на несколько парковок.

**Impact Map — влияния:** Управление ТС (и для ФЛ, и для ЮЛ); информирование; автосчета — не в первом приоритете; «решение инцидентов» → обратная связь/обращения; бронирование онлайн; бесконтактный въезд/выезд (приоритет 1); сбор данных для аналитики; сверка оплат; аудит; оплата онлайн; работа с задолженностями.

**Impact Map — поставка (Deliverables):** Конструктор договоров — не в приоритете, но учесть; CRM; оплата по QR; биллинг; ЭДО (опасение — разные сервисы у корп-клиентов); АРМ для охраны; личный кабинет; управление тарифами, шаблонами (для внутренних пользователей); сайт-визитка; георесурсы. Для 20% клиентов охранникам тоже нужны возможности в системе.

**Зоны ответственности:** Владелец — только подписание договоров с ЮЛ; всё остальное — на управляющего; владельцу нужна аналитика.

**Договорённости:** Impact Map отправить на согласование; распределение мест по категориям — заказчик скинет в чат.
