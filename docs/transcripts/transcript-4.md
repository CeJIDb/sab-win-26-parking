# Стенограмма встречи 11.02.2026

**Стенограмма встречи: User Story Map, терминология, MVP, архитектура**

**Участники:** Марина Бугаевская (ведущий), Павел Борунов (секретарь), Денис Салмин (хронометрист), Александр Друзякин, Артём Субботин; Елена Гриценко (заказчик).

**Тема:** User Story Map, согласование терминологии (Тариф, Бронирование, Договор), определение MVP, технические решения (PWA, архитектура).

---

## Ключевые темы (по ходу встречи)

- Согласование терминологии: Тариф, Бронирование, Договор
- User Story Map: блоки (тарифы и клиенты, авторизация, бронирование, договоры, оплата, въезд-выезд, отчётность)
- MVP: что входит, что за рамками (конструктор тарифа, изменение тарифа, договоры с ЮЛ)
- Технические решения: PWA, интеграция с шлагбаумами через API
- Архитектура: Business Site, Consumer App, Custom Internal Tool

---

## Фрагменты стенограммы (ключевые реплики)

**Согласованная терминология (дословно по стенограмме):** Тариф — цена товара; существует независимо от бронирования. Бронирование — покупка, заказ, фактическая резервация парковочного места; не может существовать без тарифа; актуально и для краткосрочной, и для долгосрочной аренды. Клиент оформляет услугу бронирования по выбранному тарифу; после оформления видит список бронирований, а не список тарифов. Договор — юридическое обоснование бронирования сроком больше 24 часов; для долгосрочных и ЮЛ.

**Управление тарифами и клиентами:** Просмотр тарифов — в MVP. Конструктор/калькулятор персонального тарифа — за MVP. Просмотр текущего тарифа = просмотр бронирования. Изменение текущего тарифа клиентом — за MVP. Регистрация и авторизация (в т.ч. сторонние сервисы); восстановление пароля — в MVP; редактирование профиля — опционально. Добавление и редактирование ТС — в MVP.

**Бронирование:** Выбор параметров, проверка доступности и стоимости, создание бронирования, подтверждение оплатой, отмена; время на оплату (например 15 мин); для офлайн-клиентов терминал на въезде не в приоритете. Админские истории (управление тарифами, пользователями) входят в карту.

**Решения по MVP:** Конструктор тарифа — за MVP. Самостоятельное изменение текущего тарифа клиентом — за MVP. Заключение договоров с ЮЛ — вне MVP (ручной режим). Для ЮЛ в MVP: доступ через систему, распознавание и пропуск, самостоятельное управление списком ТС, делегации. Контактные данные обязательны для всех клиентов (в т.ч. краткосрочных).

**Технические решения:** PWA для клиентов утверждено. Интеграция с шлагбаумами через API (готовые рыночные решения).

**Архитектура:** Business Site (информационный сайт); Consumer App — веб-сайт (PWA) с личным кабинетом и отдельной административной панелью для владельца и управляющего; Custom Internal Tool с CRM-функционалом.
