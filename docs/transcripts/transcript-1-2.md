## Фрагменты стенограммы (ключевые реплики)

**Новый клиент, краткосрочная парковка:** данные в таблицу, оплата, шлагбаум вручную — подтверждено.

**Новый клиент, абонемент (ФЛ):** договор, оплата (до 2 мес — полная сумма, от 2 мес — частями), выдача proximity-карты, въезд по карте — подтверждено.

**ЮЛ:** условия согласованы с управляющим; в схеме «условия согласованы» одним пунктом; оплата может быть отложенной или поэтапной; компания может начать пользоваться до фактической оплаты при договорённости; в схеме отразить, что с ЮЛ флоу отличается.

**Действующий клиент:** «старый» заменить на «действующий»; если нет действующего договора — процесс как у нового. Развилка по типу клиента: у ФЛ — краткосрочная/долгосрочная, у ЮЛ — всегда договор.

**ЮЛ на КПП:** автораспознавания нет, охранник проверяет номер по базе; если номера нет — выяснение через управляющего или представителя компании; при невозможности связи — не пропускаем.

**Карта забыта:** идентификация, проверка, шлагбаум вручную; временная карта не выдаётся.

**Карта потеряна:** новая карта за счёт клиента.

**Оплата:** счёт + чек; факт оплаты в базе договоров; охранник рассчитывает стоимость (часы × тариф). Для ЮЛ счета выставляет бухгалтер.

**Заключение договора (ФЛ):** заполнение шаблона, печать, клиент проверяет и подписывает, затем управляющий; клиенту экземпляр при следующем визите; пролонгации нет — каждый раз новый договор; паспортные данные в базе не хранятся, заполняются заново.

**Блокировка карт:** раз в неделю управляющий проверяет оплаты, при задолженности передаёт список — охранник блокирует карты в устройстве.

**Навигация:** указатели по секторам, нумерация мест; если место занято или в ремонте — клиент обращается к охраннику; часть клиентов самовольно занимает свободные места.

**Выезд:** по карте — автоматически; без карты или краткосрочный — проверка по базе, при краткосрочной проверка оплаты; при отказе от доплаты — чёрный список (ведётся по клиенту, не по номеру).

**Чёрный список:** отметка в базе клиентов, комментарии в поле примечаний.

**Инфраструктура:** осмотр охранником по заданию управляющего (шлагбаумы, освещение, камеры, покрытие); решение — своими силами или подрядчик; приём работ по акту.

**Технические уточнения:** файл учёта в Google Таблицах (синхронизация между КПП); графы «кто открыл шлагбаум» нет; при поломке ПК или нет доступа — возможна остановка приёма (только выпуск), бэкап на флешку, при офлайн — запись на бумаге или локально, затем перенос в файл. Два КПП — два отдельных места (въезд и выезд раздельно), оба работают на въезд и выезд. Перепрограммирование карты — до 10 минут; автоматической блокировки по дате нет (два состояния: активна/заблокирована).

**Пожелание заказчика:** гибкий инструмент для самостоятельного управления списком ТС корпоративными клиентами; автоматически получать из 1С данные об оплатах ЮЛ (работу бухгалтера не трогаем).
