## Фрагменты стенограммы (ключевые реплики)

**Парковка:** Санкт-Петербург, открытого типа, одноэтажная, круглосуточная; 600 мест (50 для ОВЗ). Рядом жилые дома, супермаркет со своей бесплатной парковкой, два учреждения (НИИ, статистическая служба), метро.

**Интересы владельца:** корпоративные клиенты (предсказуемость, длительный срок); краткосрочные ФЛ (высокая оборачиваемость); ФЛ на несколько месяцев/полгода.

**Тарифы:** до суток — почасовой; свыше суток — абонемент (день, неделя, месяц, несколько месяцев). Для корпоративных — индивидуальные условия.

**Персонал:** 2 КПП; на каждом — старший охранник и два линейных, смены по 12 часов; владелец и управляющий; бухгалтер на аутсорсе. IT-специалистов нет.

**Учёт:** Excel (база клиентов, договоров, журнал въезда/выезда); основная работа на охранниках и управляющем; бэкап раз в месяц на внешний носитель.

**Въезд/выезд:** шлагбаум по проксимити-карте (автоматически) или вручную; при проблемах с картой — идентификация, проверка договора и оплаты, перепрограммирование/выдача новой карты. Почасовым карту не выдают, охранник вручную пропускает. Место назначает охранник (по базе договоров); отдельной актуальной таблицы занятости нет.

**Оплата:** у охранника — наличные или терминал; предоплата; при перелимите — доплата; при досрочном выезде возврата нет (для долгосрочных — индивидуально, как правило приезд на место).

**Проблемы (озвученные Еленой):** очереди на въезде/выезде; охранники заняты не профильной работой (консультирование, оплата, договоры); нет нормальной аналитики для решений; клиенту нужно приезжать на парковку для любого действия; ~30% мест простаивают; расходы на замену карт; риск потери клиентов.

**Цели:** быстрый въезд (до 1 минуты); сокращение рутинных операций охранников с 80% до 20%; аналитика для управленческих решений. Желаемый результат: автоматизация въезда/выезда, онлайн-платформа (бронирование, оплата, тарифы), личный кабинет, аналитика для владельца, снижение простоя мест.

**Ограничения:** законное лицензированное ПО (реестр российского ПО); 152-ФЗ; авторизация через Яндекс ID, ВК ID, Госуслуги; не трогать работу бухгалтера; договоры: управляющий (по доверенности) — физлица, владелец — юрлица. Сроки: первый результат за 4 месяца, максимум 6; масштабируемость на другие парковки. Бюджет — при обосновании; итеративные поставки.

**Договорённости:** команда готовит протокол, Event Storming, список вопросов; заказчик — шаблоны Excel, уточнение опыта сотрудников по другим парковкам. Следующая встреча 23.01.2026 — детализация бизнес-процессов.
